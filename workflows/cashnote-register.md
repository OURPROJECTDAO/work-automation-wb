> 공통 개념(낱개/박스 · 합포N→배송비 · 판매가 공식 · 상품명 정제 · 카테고리 · 이미지 · 서식 일치)은
> **workflows/product-registration-common.md** 참조. 이 문서는 **캐시노트 전용값 + 검증된 전 과정**.
> 운영 = 챗 네이티브(앱 미사용, ADR 0009 채널 추가 패턴). **1차 배치 실업로드 성공 검증 완료(2026-06-17).**

## 0. 한눈에 (end-to-end)
입력(10열 시트 **또는** 업로드감시 출력) → product_master 조인(낱개매입가·박스내품·규격) → 판매가 산정(수수료6%·네이버 기준마진) → 상품명(스마트 등재명 재활용/없으면 raw 정제) → 검색어 쉼표 토큰화 → 카테고리 매핑(캐시노트 자체체계) → 상품군번호(가공식품21/비식품37) → 양식 r5부터 기입 → 재로드 검증 → 사용자 업로드.

## 1. 캐시노트 전용 값
- **수수료율 6%** (ADR 0014 — 마진모니터와 동일. 양식 인근 7% 표기는 시트 내부 불일치라 6% 채택).
- 판매가 공식·배송 경제(고객 수취 3000 / 우리 실비 2500) = 공통.
- 코드 체계: 입점사 관리코드(R) = 낱개 `PC`+상품코드(6자리 zero-pad) / 박스 관리코드(공통 규칙). **이미지 폴더는 모드 무관 항상 관리코드.**

## 2. 양식 (marketplace-bulk-product-create)
- 시트 `상품`, **42열(A:AP)**. r1=섹션 · r2=필드명 · r3=필수/선택 · r4=안내 · **데이터 r5부터**. 1~4행 삭제/변경 금지(오류). 예시행(r5 스팸세트)은 지우고 우리 것만.
- 부속 시트 3종: `카테고리 안내 참고`(가공식품 53종·전체파일은 294) · `상품정보제공고시 입력 항목 안내 참고`(상품군번호→항목) · `상품 필드값`(드롭다운 정본 — 카테고리·상품군·과세·진열 등).

### 채우는 컬럼 (14개) + 타입
| 열 | 항목 | 값 | 타입 |
|---|---|---|---|
| A | 상품명 | 정제명(박스 `{박스내품}개`/낱개 `1개` suffix) | str |
| C | 대표 옵션 | `Y` 고정 | str |
| D | 카테고리 | cashnote 카테고리 value 그대로 | str |
| H | 검색어 키워드 | 상품명→**쉼표 구분 키워드**(각 ≤20자·최대10개) | str |
| K | 판매 단가 | 공식(수수료 6%) | int |
| M | 재고 수량 | `9999` 고정 | int |
| P | 주문 N개당 배송비 | **= 합포N**(박스=합포장 / 낱개=합포장×박스내품) | int |
| R | 입점사 관리코드 | 낱개 PC+코드 / 박스 관리코드 | **str**(서식 #,##0이라도 텍스트 보존·`@`) |
| S | 과세 기준 | 과세=비움 / 면세=`면세` | str |
| T | 진열 여부 | `Y` 고정 | str |
| X | 대표 이미지 | gi.esmplus `{관리코드}_A1.{확장자}` URL | str |
| Y | 상세 이미지 | gi.esmplus `{관리코드}_B1.{확장자}` **평문 URL** | str |
| Z | 배송/반품 정책 | `DVP212991/RTP078367` 고정 | str |
| AA | 상품군번호 | **품목별** 가공식품 `21` / 비식품 `37` | int |
- **비우는 칸**: L(할인전단가/정가)·S(과세는 비움=과세)·AB~AP(고시정보값) — 전부 선택, 양식 예시도 비어있음. **비워도 가공식품 등록 정상(검증).**

## 3. 판매가 산정
```
판매가 = ⌊ (매입가 + 2500) / [(1−0.06)(1−마진율)] − 3000 ⌋   (100원 내림)
```
- 박스모드: 매입가 = 박스매입단가(업로드감시 E 또는 master). N(P열) = 합포장.
- 낱개모드: 매입가 = master `매입단가`(낱개). N(P열) = 합포장 × 박스내품.
- **마진율 = baseline_margin.csv `스마트스토어`(=네이버) 컬럼**(관리코드 키). **없으면 10%**(사용자 확정 2026-06-17). 캐시노트 컬럼이 아니라 네이버 기준 — 이미 스마트 등재 상품의 확정 마진을 그대로 캐시노트에 적용.
- 매입단가 소수값 가능(예 5092.5) → float 처리.

## 4. 상품명 (A) — 등재명 재활용
- **이미 등재된 채널의 listing 스냅샷에서 정제명 재활용 = 이름 재가공 리스크 0.**
- 1순위: `reference/listing_smartstore.csv` 코드조인(박스=관리코드 / 낱개=`PC`+상품코드 **6자리 zero-pad**, 예 PC005900). 등재 상품이면 정제명·suffix(`{박스내품}개`) 그대로.
- 미매칭(스냅샷에 없음): raw명 자동정제 — 끝 ` (NN)`·`*NN` 제거 + suffix. ⚠️ 띄어쓰기 등 거칠 수 있어 사용자 확인 권장.

## 5. 검색어 키워드 (H, 필수) ★실패교훈
- 캐시노트 H = **쉼표 구분 · 각 키워드 ≤20자 · 최대 10개**. **상품명 통짜 1개 키워드 금지** — 20자 초과 시 "최대 길이를 초과했습니다" 거부(1차 배치 10건 실패→픽스 성공).
- 생성: 상품명(단위 suffix 제거) → `re.split(r'[\s()\[\]/+,·]+')` 토큰화 · 각 ≤20자(초과 절단) · 중복제거 · 최대 10개 · 쉼표 결합.
- 예 `칸타타 파우치 커피 스위트 아메리카노 제로 230ml` → `칸타타,파우치,커피,스위트,아메리카노,제로,230ml` / `샘표 양조간장501(1.7L+양조500ml)` → `샘표,양조간장501,1.7L,양조500ml,6세트`.

## 6. 합포 → 배송 (스마트스토어와 다름)
- 캐시노트는 **배송비템플릿코드표 없음**. **P열 '주문 N개당 배송비 부과'에 합포N 직접 기입**.
- 실제 배송비 금액은 Z 배송정책코드(DVP212991→3000)가 결정. Z=`DVP212991/RTP078367`(배송정책/반품교환정책) 고정쌍.

## 7. 카테고리 (D) + 상품군번호 (AA)
- 캐시노트 자체 체계(스마트스토어/ESM 코드와 별개). 매핑 = 상품→코드 AI 제안(+필요시 웹검색) → 사용자 확정. **되는 것만 매핑·불명은 빈칸(사용자가 채움).**
- 표:
  - `reference/cashnote_category_food.csv` (가공식품 53종) — 주력.
  - `reference/cashnote_category_all.csv` (**전체 295종** — 가공/신선식품·주방용품·라이프·배달/사무/렌탈). 비식품 섞인 배치용.
  - 컬럼 code·l1·l2·l3·**value**. D = **value 그대로**(`[코드] 1차 / 2차 / 3차`). 공백·표기 불규칙(예 010101 `냉장/냉동/ 햄`)해도 정본 그대로(드롭다운 일치). value=양식 드롭다운 정본(전체표 4종은 식품계정 드롭다운 외라 재구성).
- **상품군번호(AA) = 품목별** (고시 시트 군번호로 결정, 고정 아님):
  - 가공식품(카테고리 01xxxx) → **21**.
  - 비식품 → **37 생활화학제품**(세제·세정제·섬유유연제·락스·물티슈 등). 기타 군번호: 18 화장품·17 주방용품·20 식품(농수축산)·38 살생물제품.
  - ⚠️ 물티슈는 잠정 37(생활화학) — 인체세정용=18 화장품·주방일회용=17 주방용품일 수 있음(사용자 판단).
- 매핑 규칙(1차 배치 실측): 간장/춘장=010501 · 참치/골뱅이/번데기/꽁치/런천/고등어=010802 캔통조림 · RTD차(헛개·보리·녹차·우롱·유자)=011001 · 주스/탄산/에이드원액 음료=011006 · 에이드/농축액=011004 · 시럽=011002 · 스낵/칩/꽈배기/누룽지=010803 · 설탕010404·소금010405·콩식용유010401·참기름010403·고추맛기름/맛기름010402 · 향신료(오레가노/바질)010504 · 파스타/마카로니010703 · 락교/피클=010903 절임 · 주방세제080501·세탁세제080502·세정제(락스/펑크린)080504·물티슈040403.

## 8. 이미지
- 대표(X): `https://gi.esmplus.com/td680708/{관리코드}/{관리코드}_A1.{확장자}`
- 상세(Y): `…_B1.{확장자}` **평문 URL** — ⚠️ 스마트스토어/ESM과 달리 `<img src=...>` 래핑 **없음**.
- 확장자 jpg→png 실검사(공통). 업로드감시 출력은 확장자(K/N) 이미 해소돼 URL(L/O) 그대로 사용 가능.

## 9. 입력 경로 (2가지)
- **10열 표준 입력시트**(스마트스토어 공용): 상품코드·관리코드·상품명·박스/낱개·최대합포·목표마진·면세/과세·카테고리·대표/상세이미지URL.
- **업로드감시 출력 XLSX**(권장·1차 배치 사용): 등록 필드 보유(상품코드·관리코드·상품명·모드G·합포H·박스매입가E·이미지URL+확장자L/O·K/N·채널상태P/Q) → 10열 시트 생략하고 직접 구동. ⚠️ 카테고리(I, 스마트 기준)는 비어옴 → 캐시노트 체계로 재매핑. 낱개매입가·박스내품은 master 조인 보충. 마진율=baseline 네이버 컬럼.

## 10. 자산 위치 (고정 — 챗 네이티브)
- **양식(빈)**: app repo `reference/cashnote_bulk_template.xlsx` (상품 42열·1~4행·부속 3시트). 배치 시 raw 다운로드→openpyxl load(read_only 금지)→r5 예시 지우고 기입.
- **카테고리표**: `reference/cashnote_category_food.csv`(53) + `reference/cashnote_category_all.csv`(295).
- **마진/매입/이름**: `reference/baseline_margin.csv`(스마트스토어 컬럼)·`reference/product_master.csv`(매입단가/박스매입단가/박스내품/규격)·`reference/listing_smartstore.csv`(등재 정제명).
- **입력폼**: 스마트스토어 10열 표준 시트 공용.

## 11. 생성 절차 (검증된 순서)
1. 양식 load(read_only 금지) → 42열·1~4행·셀서식 보존.
2. 입력(10열 또는 업로드감시 출력) 로드 → product_master 상품코드 조인(낱개매입가·박스내품·규격).
3. 판매가 산정(매입가·네이버 기준마진·없으면 10%·공식·100원내림). N = 박스 합포 / 낱개 합포×박스내품.
4. 상품명 = listing_smartstore 재활용(미매칭 raw 정제). 검색어 = 쉼표 토큰화(각 ≤20자).
5. 카테고리 매핑(되는 것·불명 빈칸) → 상품군번호(가공식품21/비식품37).
6. r5부터 14열 기입. **R은 텍스트(`@`)**·K/P/M/AA 숫자·S 과세는 비움·L/AB~AP 비움.
7. 예시행 덮어쓰고 우리 N행만. 재로드 검증(시트4·열42·1~4행·행수·R 텍스트·AA 분포·카테고리 빈칸수).

## 13. 검증된 사실 (1차 배치)
- ✅ **openpyxl 저장본 캐시노트 업로더 수용**(데이터검증/드롭다운 extension 제거돼도 등록 정상) — **zip레벨 수술 불요**(쿠팡류와 다름).
