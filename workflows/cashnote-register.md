# cashnote-register (캐시노트 = KCD 마켓플레이스 상품 등록) — 플랫폼 인스턴스

> 공통 개념(낱개/박스 · 합포N→배송비 · 판매가 공식 · 상품명 정제 · 카테고리 · 이미지 · 서식 일치)은
> **workflows/product-registration-common.md** 참조. 이 문서는 **캐시노트 전용값**만.
> 운영 = 챗 네이티브(앱 미사용). 입력 = 스마트스토어 10열 표준 시트 공용(카테고리만 캐시노트 체계).

## 캐시노트 전용 값
- **수수료율 6%** (ADR 0014 — 마진모니터와 동일. 양식 K열 인근 7% 표기는 시트 내부 불일치라 6% 채택).
- 판매가 공식·배송 경제(고객 3000 / 실비 2500) = 공통. 분모 = (1−0.06)(1−마진율), 100원 내림.
- 코드 체계: 입점사 관리코드(R) = 낱개 `PC`+상품코드 / 박스 관리코드(공통 규칙). **이미지 폴더는 모드 무관 항상 관리코드.**

## 양식 (marketplace-bulk-product-create)
- 시트 `상품`, **42열(A:AP)**. r1=섹션 · r2=필드명 · r3=필수/선택 · r4=안내 · **데이터 r5부터**. 1~4행 삭제/변경 금지(오류). 예시행(r5 스팸세트)은 지우고 우리 것만.
- 부속 시트: `카테고리 안내 참고`(가공식품 53종) · `상품정보제공고시 입력 항목 안내 참고`(상품군별 항목) · `상품 필드값`(드롭다운 정본 294종).
- 채우는 **~13열** + 셀 타입:

| 열 | 항목 | 값 | 타입 |
|---|---|---|---|
| A | 상품명 | 정제명(박스 `{박스내품}개`/낱개 `1개` suffix, 스마트스토어 동일) | str |
| C | 대표 옵션 | `Y` 고정 | str |
| D | 카테고리 | cashnote_category_food value 그대로 | str |
| H | 검색어 키워드 | **상품명에서 단위 suffix 제거** | str |
| K | 판매 단가 | 공식(수수료 6%) | int #,##0 |
| M | 재고 수량 | `9999` 고정 | int |
| P | 주문 N개당 배송비 | **= 합포N** (박스=합포장 / 낱개=합포장×박스내품) | int |
| R | 입점사 관리코드 | 낱개 PC+코드 / 박스 관리코드 | **str**(45-11형·#,##0서식이라도 문자 보존) |
| S | 과세 기준 | 과세=비움 / 면세=`면세` | str |
| T | 진열 여부 | `Y` 고정 | str |
| X | 대표 이미지 | gi.esmplus `{관리코드}_A1.{확장자}` URL | str |
| Y | 상세 이미지 | gi.esmplus `{관리코드}_B1.{확장자}` **평문 URL** | str |
| Z | 배송/반품 정책 | `DVP212991/RTP078367` 고정 | str |
| AA | 상품군번호 | `21`(가공식품) 고정 | str/int |

## 고정값
- C 대표옵션=Y · M 재고=9999 · T 진열여부=Y · Z=`DVP212991/RTP078367`(DVP212991=유료배송 3000·마진모니터 확인값) · AA 상품군번호=`21`(가공식품).
- **L 할인전단가(정가) · AB~AP 고시정보값 = 비움** (양식 예시도 비어있음·선택, 사용자 확정 2026-06-17).
- S 과세: 과세=비움(기본) / 면세=`면세` (입력시트 면세/과세).

## 합포 → 배송 (스마트스토어와 다름)
- 캐시노트는 **배송비템플릿코드표 없음**. **P열 '주문 N개당 배송비 부과'에 합포N 직접 기입**.
- N = 박스모드 합포장 / 낱개모드 합포장×박스내품 (공통).
- 실제 배송비 금액은 Z 배송정책코드(DVP212991→3000)가 결정.

## 카테고리 (D)
- 캐시노트 자체 체계(스마트스토어/ESM 코드와 별개). **reference/cashnote_category_food.csv** (53종 가공식품). 컬럼 code·l1·l2·l3·**value**.
- D = **value 그대로**(`[코드] 1차 / 2차 / 3차`). 공백·표기가 불규칙(예 010101 `냉장/냉동/ 햄`)해도 정본 그대로 박을 것(드롭다운 일치).
- 매핑: 상품→코드 AI 제안 + 필요 시 웹검색 → 사용자 확정(스마트스토어 동일). 비식품(라이프 0805xx 등) 필요 시 양식 `상품 필드값`(294종)에서 추가.

## 검색어 키워드 (H, 필수)
- = **상품명 그대로(단위 suffix 제거)**. 정제 상품명에서 끝 ` 1개`/` {박스내품}개` 떼서 H에 기입. (사용자 확정 2026-06-17)

## 이미지
- 대표(X): `https://gi.esmplus.com/td680708/{관리코드}/{관리코드}_A1.{확장자}`
- 상세(Y): `…_B1.{확장자}` **평문 URL** — ⚠️ 스마트스토어/ESM과 달리 `<img src=...>` 래핑 없음.
- 확장자 jpg→png 실검사(공통). gi.esmplus.com 접근 가능.

## 데이터 출처
- product_master(app repo reference): 관리코드/상품코드 → 박스내품·박스매입단가·낱개매입단가·규격.
- 정제명/모드/합포장/카테고리/과세 = 10열 입력시트.

## 자산 위치 (고정 — 챗 네이티브)
- **양식(빈)**: app repo `reference/cashnote_bulk_template.xlsx` (상품 42열·1~4행 보존·부속 3시트). 배치 시 raw 다운로드→openpyxl load(read_only 금지)→r5 예시 지우고 기입.
- **카테고리표**: app repo `reference/cashnote_category_food.csv` (53종).
- **입력폼**: 스마트스토어 10열 표준 시트 공용.
- (앱 코드 참조 0건 — 대화창만 사용.)

## 생성 절차
1. 양식 load(read_only 금지) → 42열·1~4행·셀서식 보존.
2. master 조인 → 판매가/N/이미지확장자 산정 → 카테고리 AI매칭(애매=상의) → 상품명 AI정제(사용자 확정).
3. r5부터 기입. **R(관리코드)은 문자로**(45-11형, #,##0 서식이라도 텍스트 보존). K/P/M 숫자.
4. 예시행 덮어쓰고 우리 N행만. 재로드 검증(시트·열수 42·1~4행·행수·R 텍스트).

- 10열 표준 입력시트(스마트스토어 공용).
