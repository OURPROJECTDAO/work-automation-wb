# sikbom-register (식봄 = 마켓보로 식자재몰 상품 일괄등록) — 플랫폼 인스턴스

> 공통 개념(낱개/박스 · 합포N→배송비 · 판매가 공식 · 상품명 정제 · 카테고리 · 서식 일치)은
> **workflows/product-registration-common.md** 참조. 이 문서는 **식봄 전용값 + 검증된 전 과정**.
> 운영 = 챗 네이티브(앱 미사용, ADR 0009 채널 추가 패턴). **1차 41건(74→디듑33) 실업로드 성공·검증 완료(2026-06-18). 운영중.** ✅ openpyxl 저장본 식봄 수용 확인.
> ※ 식봄은 마진모니터·송장처리·행사기획엔 기존 채널이나 **상품 일괄등록은 이번이 최초**.

## 0. 한눈에 (end-to-end)
업로드감시 3채널 출력(식봄='업로드필요' 필터) → product_master 조인(낱개매입가·박스내품·규격) → 판매가 산정(수수료7%·스마트스토어 기준마진) → 상품명 정제(listing 재활용 0매칭이라 raw 정제) → 카테고리 매핑(sikbom_category 명 그대로) → 배송비명 L 산정(N규칙) → 키워드 토큰화 → 양식 '상품 대량 등록' r8부터 기입 → 재로드 검증 → 사용자 업로드.

## 1. 식봄 전용 값
- **수수료율 7%** (channel-margin-monitor 식봄과 동일).
- **마진율 = baseline_margin.csv `스마트스토어`(네이버) 컬럼**(관리코드 키)·**없으면 10%**(사용자 확정). 캐시노트와 동일 — 식봄 컬럼 아님(이미 스마트 확정 마진 재활용).
- 판매가 공식·배송 경제(고객 수취 3000 / 우리 실비 2500) = 공통.
- 코드 체계(A 상품코드 = ERP코드): 낱개 `PC`+상품코드(6자리 zero-pad) / 박스 관리코드. **이미지 폴더는 모드 무관 항상 관리코드.** (A는 이미지 대량등록 매칭키라 필수권장)
- **과세 = 전건 y(과세)** 디폴트(따로 지정 없으면, 사용자 확정).

## 2. 양식 (상품_대량등록 양식·ver 2026-02-26)
- **시트 2개**: `상품 대량 등록`(29열 A:AC) + `상품 카테고리`(드롭다운 소스·식봄 원본 367 트리플 — **건드리지 않고 보존**).
- `상품 대량 등록`: r1 유의사항 · r2~4 안내 · r5 필수도 · **r6 필드명(헤더) · r7 안내 · 데이터 r8부터**. r1~7 삭제/변경 금지. 예시행(r8 르씨엘)은 지우고 우리 것만.
- **이미지 컬럼 없음** — 식봄은 별도 '상품이미지 대량등록(상품코드 A 매칭)' 방식. 양식엔 이미지 안 넣음. **이미지 = 업로드 성공 후 별도 단계(phase 2).**

|---|---|---|---|

## 3. 판매가 산정 (공통 §3 = 캐시노트와 동일, 수수료만 7%)
```
판매가 = ⌊ (매입가 + 2500) / [(1−0.07)(1−마진율)] − 3000 ⌋   (100원 내림)
```
- 박스모드: 매입가 = 박스매입가(업로드감시 H 또는 master 박스매입단가). N = 합포단위.
- 낱개모드: 매입가 = master `매입단가`(낱개). N = 합포단위 × 박스내품.
- 정가 O = 무늬 가짜(권장가*1.25·10원단위·Q≤O). 실정가 있으면 그걸로.

## 4. 상품명 (E)
- listing_sikbom 등재명 재활용 시도하나 **신규(업로드필요)는 식봄 미등재 → 0매칭** → raw명 자동정제 + 단위 suffix.
- 정제 = 띄어쓰기·브랜드 앞·내부코드/(NN)·`_`·`/` 정리 + 용량 유지. **AI 제안 → 사용자 확정**(검토 워크시트).

## 5. 카테고리 (B/C/D)
- 식봄은 **코드 없이 카테고리명 그대로** B(1차)/C(2차)/D(3차)에 기입.
- 표 = `reference/sikbom_category.csv`(1차/2차/3차 평탄·UTF-8-sig). 원본 양식 '상품 카테고리' 시트가 **2축 블록 매트릭스**(1차마다 블록·2차 가로헤더·3차 세로)라 평탄화한 정리본. 사용자 큐레이션 266 트리플(취급분만·12개 1차).
- 매핑 = 상품→leaf AI 제안(필요시 웹검색) → 사용자 확정. **양식 업로드용은 식봄 원본 '상품 카테고리' 시트(367·드롭다운 소스) 보존** — 우리 266은 내부 매핑용 서브셋(전부 원본의 유효 leaf).

## 6. 합포 → 배송비명 (L) ★식봄 고유
- 식봄은 배송비코드/템플릿 없음. **L = 식봄 판매자센터 [택배 배송비 설정]에 등록된 '배송비 명' 텍스트** 그대로.
- 규칙(골든 예시 PC004307 낱개·박스내품12·합포1 → N=12 → `수량별 배송비 12`로 확정):
```
N = 박스모드 ? 합포단위 : 합포단위 × 박스내품   (= 공통 N)
L = N==1 ? "유료배송(수량별)" : f"수량별 배송비 {N}"
```
- ⚠️ **운영 전제: 쓰이는 N마다 `수량별 배송비 {N}` 배송비명이 식봄 센터에 미리 등록돼 있어야 통과**(없으면 그 행 거부). 양식 생성 시 사용 배송비명 목록을 사용자에게 제공.

## 7. 키워드 (AC, 선택)
- **쉼표 구분 · 각 ≤15자 · 최대 10개 · 한글/영어/숫자만**(특수문자·`.`·`/` 불가 — 캐시노트 ≤20자보다 엄격).
- 생성: 정제명 토큰화(`[\s/()\[\]+,·×*]+`) → 한글/영어/숫자만 잔류 → 각 ≤15자 절단 → 중복제거 → 최대 10개 → 쉼표. (`1.8L`→`18L`로 절삭됨 — 허용)

## 8. 입력 경로 (1차 배치)
- **업로드감시 3채널 출력 XLSX**(이미지 포함·18열): A상품코드·B관리코드·C상품명·D박스/낱개·**E합포단위**·F박스재고·**G면세과세**·H박스매입가·이미지URL/확장자·P스마트스토어/**Q식봄**/R캐시노트 상태. **Q(식봄)='업로드필요' 필터**가 등록 대상.
  - ⚠️ 이 출력엔 합포·과세가 컬럼으로 포함(구 캐시노트 출력과 레이아웃 다름). 없을 땐 합포단위는 사용자 입력 필요(기존 채널 리스팅 역산 0/74 — 못 끌어옴).
  - 낱개매입가·박스내품·규격은 master 조인 보충. 마진율=baseline 스마트스토어.

## 8b. 이미지 (phase 2 — 양식과 별개·업로드 성공 후) ✅
- 식봄 이미지 대량등록 = **상품코드 폴더 구조 zip**. 폴더·파일명 = **식봄 등록 상품코드(양식 A)**: 폴더 `{상품코드}`, 대표 `{상품코드}_1.{ext}`, 상세 `{상품코드}_d1.{ext}`.
- 우리 소스 = `gi.esmplus.com/td680708/{관리코드}/{관리코드}_A1.{ext}`(대표)·`_B1`(상세). **A1→_1, B1→_d1 리네임**. ⚠️ 폴더/파일=상품코드(A) / 소스=관리코드 — **낱개는 둘이 다름**(PC+코드 vs 관리코드)이라 리네임이 실질 변환.
- 추출 아님 — 호스트에서 다운로드. 업로드감시 출력의 대표/상세 URL·확장자(K/N·L/O) 그대로 사용(실검사 ext 해소됨·gi.esmplus 접근 가능·HTTP200).
- 절차: 업로드감시 출력 URL → 각 다운로드 → `{A}/{A}_1.{ext}`·`{A}/{A}_d1.{ext}` → zip. 1차 41건=82장 전건 성공(대표 1000×1000·상세 860×가변). 산출 `식봄이미지_41건.zip`.

## 10. 생성 절차 (검증된 순서)
1. 양식 복사 → load(read_only 금지) → 2시트·r1~7·셀서식 보존.
2. 업로드감시 출력 로드(식봄='업로드필요' 필터) → master 조인.
3. 기계적 산정: A코드·매입가·N·배송비명L·마진·판매가Q·정가O(Q≤O)·과세Z·키워드AC·고정값.
4. AI 제안: 정제명 E·판매규격F/단위G·카테고리B/C/D → **검토 워크시트로 사용자 확정**(노란칸 수정·회색칸 잠금).
5. 확정본으로 r8부터 기입. A·AC는 텍스트(`@`). r8 예시 덮어쓰고 우리 N행만.
6. 재로드 검증(시트2·29열·r1~7·행수·Q≤O·배송비명·키워드 15자/10개·과세).
