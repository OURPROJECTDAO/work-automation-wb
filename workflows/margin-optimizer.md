## 0. 한 줄
P×C(상품 ~600 × 채널 8 ≈ 5,000)마다 **최적 기준마진율**을 데이터로 잡아 채널마진모니터 편집창에 권장값으로 prefill. 사람이 못 만지는 5천 개를, 우리가 쥔 단 하나의 레버(기준마진율 시트)로 순이익 최대화하도록. 두뇌③(탄력성 서술)을 처방으로 올린 단계 = 두뇌④.

## 1. 목표 / 원칙 (사용자 확정)
- **최대화 대상 = 순이익(원).** "괜찮은 마진 범위"면 매출 볼륨 우선.
- **하한 = 나들커뮤니케이션 마진율.** 가장 공격적인 B2B 마트 채널 = 가격 바닥 기준. 처음부터 거기까지 갈 필요는 없음(이미 볼륨 깔아주는 채널이 있으니).
- **운용 인력 = 현재 무시**(나중 고려). 지금은 매출·순이익 볼륨 우선 작전.
- **채널 독립 = 새 수요로 본다**(사용자 확정 Q3=A). 채널끼리 손님이 달라 → 한 채널 인하가 타 채널을 잠식하지 않음 → **채널별로 따로 최적화 OK**(교차잠식 모델링 불요).
- **순이익 꼭짓점에서 멈춤**(Q4): 순이익 최대 지점이 "가장 좋은 마진". 볼륨 더 가지려 순이익 양보 안 함. ★단 꼭짓점이 현재 마진의 좌/우 어디인지는 상품마다 다름 = 데이터로 풀 문제(ε×마진>1인 동안만 인하가 순이익↑).

## 2. 베이스 (상품 적정 마진) — 척추
- **베이스 = 순이익 누적 ~85% 차지하는 proven 채널들의 *순이익 가중평균* 마진.** (단순평균 아님 — 순이익이 목표라 큰 채널 쪽으로 기움. Q9=가, Q10=가+가중)
- proven 채널 경계 = **순이익 상위 누적 ~85%**(절대 개수/볼륨 임계 아님 — 상품마다 단가·물량 천차만별이라 상대 기준). 검증 15-04: 식봄+캐시노트+스마트 = 순이익 96% → 이 3개, 베이스 ≈ 6.4%.
- proven이 1개뿐이면 → 베이스=그 채널(평균 불요). 게이트 🟡(불안정).

## 3. 채널별 행동 — 볼륨 ×(마진 vs 베이스) 4분면
- **고볼륨(proven)**: 베이스를 *정의*함. 유지.
- **저볼륨·고마진(>베이스)**: 가격이 높아 안 팔리는 가설 → 베이스로 **절반스텝** 끌어내림(노브1). (검증 15-04 알리 11%·배민 8%)
- **저볼륨·저마진(<베이스)**: 이미 싼데도 안 팔림 → 가격 문제 아님 → **hold-low**(낮게 유지, 올리지 않음). 팔리기 시작하면 재평가. (검증 15-04 ESM 4.9%)
- ⑤ 4분면이 *절대 마진*이 아니라 **베이스 대비**로 도는 게 핵심.

## 4. 탄력성 판단 (시장가 대신 채널 내부 비교)
- 같은 상품의 채널 간 (고객가, 볼륨) 편차에서 가설을 빌려옴(⑥). **더 싼 채널이 더 잘 팔리는 게 아니면 가격이 제약이 아님 → 더 안 내림.** (검증 15-04: 스마트 −0.5%(시장가 대비)가 식봄 +1.9%보다 더 팔리는 게 아님 → 가격 인하 불요 → 적정 6.4%.)
- 마진을 **흔든 이력**이 있으면 그 셀의 마진↔볼륨으로 데이터 최적. 없으면(평탄·corr≈0, 예 31-04-02) **시도→측정 루프**(실험)로 발견. 관찰 데이터의 인과 한계를 실험 루프가 메움(= Gate 2·3 작동).

## 5. 노브 4종 (확정)
1. **저볼륨·고마진 끌어내림 = 절반 스텝.** 베이스까지 거리의 1/2씩, 다음 사이클 볼륨 반응 보고 또 절반 / 안 늘면 되돌림. 한 방 인하·외삽 금지(그 채널 탄력성 모름 — 가격 탓인지 수요 부재인지 미지).
2. **회전(②) 인하 = 심각도 비례 + 캡 + 자동복귀.** 트리거=소진예측일 길수록(두뇌② 사장재고/장기). 한 사이클 **−2%p 캡**으로 점진. 재고 풀리면 회전 정상화 → 베이스로 자동 복귀. 바닥=역마진 금지 + margin_floor(제조사 단가하한). ★베이스 아래로도 일시 허용(청산). 단방향(회전 빠르다고 ↑ 안 함).
3. **게이트 = 전부 사람 저장 거침 + 3단계 플래그.** 라이브 자동반영 없음(Gate 1 = 사람은 결정만). 🟢수락(작은 절반스텝·신호 신뢰·충돌 없음, 일괄 처리) / 🟡검토(proven 1개뿐·중간 변동) / 🔴필수(①라벨 충돌 ②|Δ|>3%p ③매출목표·시즌 상품 ④신호 없음=실험 후보).
4. **관망 임계 = 비중 <1%는 *제외* 아니라 *저우선 실험큐*.** 소형 채널도 가격 테스트 1번은 줌(가격이 막았는지 관찰) → **2회 무반응이면 park(쉼)**. park도 영구 아님(분기·원가·시즌·시장 변동 시 자동 재시도). **신규 상품/채널은 항상 실험 대상**(이력 없음 → 관망 불가). ★"관망"은 "작다"가 아니라 "테스트 무반응" 상태(콜드스타트 채널 영구배제 방지).

## 6. 라벨 로스터
**채택 — 자동(데이터가 붙임)**
- ⑤ 마진×볼륨 4분면(베이스 대비) — 뼈대. 신규 상품 빠른 기준 탐색 포함.
- ⑥ 채널 편차 — 탄력성 가설(비싸도 잘 팔리는 채널 = 가격 둔감 → 마진 유지/↑).
- ② 재고 회전 — 장기소진만 마진↓(단방향, 노브2).

**채택 — 사람(≤600 관리코드에 라벨)**
- ⑦ 상품 매출 목표 — 최소 매출 보장 상품. 미달→볼륨 푸시(마진↓)·초과→마진 챙김.
- ⑧ 시즌/명절 — 선물세트 등 기존 라벨 재사용. 평시 탄력성 분석 제외·시즌 별도 정책.

**보류**
- ① 시장가(nadl) — ESM 등 쿠폰 왜곡으로 우리 역산 gross ≠ 고객 실결제가. 일단 빼고 sell-through(순마진+볼륨)로 해석. **복귀 선결 = 고객 실결제가↔nadl 비교 정의**(market_nadl 매핑은 이미 있음).
- ④ 재구매율 / ⑨ 미끼(대표상품) — **선결 = 채널×상품 고객 데이터 구조화**(고객키/합포박스키는 적재됨 ADR 0021, 채널×상품 재구매 집계는 미정).

**제외 (라벨 아님 — 대시보드 알림만)**
- ③ 원가추세(수정로그·2C) — 일시적이라 라벨 부적합. 두뇌①·데일리 가격변동알림이 이미 커버.

## 7. 결합 메커니즘
- **(b) 베이스+가감 + (c) 충돌만 추천탭.** 제품 단위 라벨(②⑦ + 베이스)이 가드레일/전반 자세를 깔고, 채널 단위 라벨(⑤⑥)이 채널마다 ±조정. 진짜 충돌(예: 회전 김→내려라 vs 이미 시장 최저→가격 문제 아님)은 자동 말고 🔴 추천탭에 띄워 사람 판단.
- **(a) 단일 서열은 부적합**(실증): 라벨이 서로 다른 그레인(①②⑦=제품 단위 / ⑤⑥=채널 단위)·다른 측정이라 한 줄로 못 세움.

## 8. 출력 / 루프
- 출력 = (관리코드, 채널)별 **권장 기준마진율** + 플래그(🟢🟡🔴) + 이유(베이스·4분면·회전 등).
- → 기존 **channel-margin-monitor 기준마진율 편집 data_editor에 prefill**(채널 baseline_col만 수정·라이브 read 즉시반영, ADR 0016 재사용). 사람 확인 후 저장.
- → 저장 시 listing 가격변경(절반스텝 권장가) 양식 생성(cmm 가격변경 빌더 재사용).
- → 다음 사이클: 1d listing 스냅샷·매출자료로 **반응 측정**(볼륨 변화) → 유지/추가 인하/되돌림. = 시도→측정→학습 루프(Gate 2·3).

## 9. 데이터 (전부 적재됨 — 재적재 0)
- 매출(정산 진실)·순이익·볼륨 = master/sales_YYYY-MM.parquet(매출자료). 마진율=순이익/정산액(기준마진율 시트 정의).
- 실측 택배 = ship_alloc(8 EA 채널 송장 실배분·합포 ceil(팩/3)·00-12 정합). 비-EA 채널(나들 등)은 채널 00-12 총택배 정확·품목 분배는 물류량(수량÷박스내품) 추정.
- 회전 = 두뇌② stockout(현재고÷소진율·입고주기 cadence).
- 베이스/대비 = channel_compare(두뇌③ compute_online_margin·channel_breakdown 재사용).
- 시장가(보류) = market_nadl(nadl_map.csv·prices_*.parquet).
- 기준/하한 = baseline_margin.csv·margin_floor.csv.

## 10. 전용 함정
- **마진 정의 = 기준마진율 시트와 동일**(순이익/정산액, 실택배비 2700/박스). 매출자료 `판매이익`은 택배 전 gross → 반드시 ship_alloc 택배 빼서 순이익으로. (분모를 매입가로 두면(두뇌③ A/B 정의) 값이 달라짐 — 시트 prefill엔 정산액 분모.)
- **비-EA 채널 품목 마진은 추정**(나들 등 ship_alloc 미적용 → 물류량 비례 배분). 채널 총마진은 정확. 박스내품 master 값 정확도 의존(불일치 시 그 품목 택배 왜곡 — 예 코카콜라355 master 박스내품 24 vs 상품명 88).
- **ESM 등 쿠폰 채널 = 우리 역산 gross ≠ 고객 실결제가** → 시장가 비교 왜곡. ESM은 "비싸다"가 아니라 "순마진 5%인데도 안 팔린다"로 해석(sell-through).
- **관찰 데이터 = 상관≠인과.** 과거 실적 서술은 신뢰. "마진 올리면/내리면 ~된다"의 미래·가정은 실험 루프로 확정(행사·계절·품절·성장추세 교란). 성장추세가 corr를 과장(고마진=초기 소물량·저마진=최근 대물량).
- **관망 ≠ 영구배제**(노브4) — 콜드스타트 채널/신규 상품을 영영 못 키우는 함정 방지. 2회 테스트 무반응만 park, 주기 재시도.

## 11. 검증 (설계 단계 — 실데이터 인터뷰)
- 15-04 스팸클래식340(nadl 매칭): 채널 7개 풀 라벨 산출. 순이익 96% 상위 3채널, 베이스≈6.4%, 액션=캐시노트↑ 1칸. ESM(+23.5% gross·물량 64)=쿠폰 왜곡→hold-low.
- 31-04-02 코카콜라500 스마트: 18개월 마진 평탄·corr≈0 → 신호 없음 = 실험 대상(게이트 정상 작동).
- 나들커뮤니케이션: B2B 마트 채널·택배 4.64억/12개월 → 총마진 10.6%→순마진 4.0%(하한 채널의 실체).

### 마진 정의 정합 (ADR 0027) — 중요
- cmm 기준마진율 = **노출가(표시가) 기반 forward target**(정산=노출가×(1−수수료)+배송비×계수). 두뇌④ = **매출자료 realized**(정산=실제 정산금액, 쿠폰·할인·믹스 반영). **같은 공식, 다른 정산액.**
- 매입가 ≈ 동일(product_master). 택배: cmm 플랫(2700/박스내품) vs 두뇌④ ship_alloc 실측 → 약간 다름.
- 실데이터(15-04 캐시노트): cmm 타깃 8.56% vs 두뇌④ 실현 5.74% = **실현갭 2.82%p**(대부분 정산 차이).
- ★**권장(realized)을 cmm 타깃에 절대값으로 쓰면 방향 역전**(실현 하락). → **기준마진율 변경 = 현 타깃 + Δ**(Δ=권장−현재). 기준값 없는 행은 기록만. 작은 절반스텝 전제(할인갭 ≈ 일정).
- 표시: 기준마진율(현 타깃) 컬럼(load_baseline) · 채널→baseline 컬럼 매핑 _BCOL(ESM(G마켓·옥션)→ESM).
