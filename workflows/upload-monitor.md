# 워크플로우: upload-monitor (업로드감시 — 채널 등록 갭/품절 정합)

> 이 워크플로우를 건드리기 전 이 파일을 읽는다. 전역 함정은 pitfalls.md, 공유자산 연결은 manifest.md.

## 요약
- **박스재고가 있는데 각 채널에 아직 등록(업로드) 안 된 관리코드**를 탐지하는 모니터. (역으로 재고0인데 채널엔 살아있는 = 품절처리 대상도 같이.)
- 출처 골든 = `스마트스토어 마진율 계산기 V12.xlsm`의 `★재고관리시트`(외부참조 수식·12채널 이름 COUNTIF·감시대상 수동 추림 비연속행). 이를 **코드기반 자동 매칭 + 등록 인계 + 대시보드 커버리지**로 발전.
- 허브 위치: 발주(logistics-order, 재고 진실) ─재고─▶ **업로드감시** ─갭─▶ 등록(register). 옆으로 마진모니터와 listing·resolve_code·product_master 공유.
- 상태: **코어+독립 페이지 배포, L4 대기.** (2026-06-12, ADR 0017) ⚠️ core 신규 모듈 첫 배포 → Reboot app 1회.

## 핵심 통찰 — base 정체로 collapse
- 낱개(PC+상품코드)·박스(관리코드)·소분(변환-원코드)·합포(-CB-)가 **전부 같은 product_master 행(상품코드)으로 환원**된다(채널마진모니터 `resolve_code` 4-tier 재사용).
- → "낱개/박스/소분/합포 중 **하나라도** 업로드면 그 관리코드는 업로드됨"이 **base 정체로 키 잡으면 자동 성립**(별도 OR 불요). 골든의 스마트스토어 `관리코드명 OR 소분코드명` 땜빵을 전 채널 일반화·대체.
- 이름(COUNTIF) 매칭 → **코드 매칭**이라 NFC/이름변형 취약점 제거.

## 정규화 키 = 상품코드 (★실물검증 2026-06-12)
- product_master 4,340행. **상품코드[3] = 유니크 키(빈값0)**. 관리코드[4]는 빈값114 + 충돌2(`27-08`,`27-01-05`→상품코드 2개) → 키 부적합.
- 조인/dedup = **상품코드**. 표시 = 관리코드(박스코드). `load_references`의 `pm_by_mgmt`는 setdefault(충돌 first-win, 무시 가능).

## product_master 컬럼 (0-index, utf-8-sig)
상품코드[3]·관리코드[4]·**상품명[5]**·규격[6]·박스내품[7]·매입단가(낱개)[8]·박스매입단가[9]·최종재고[13]·**박스재고=[14]("박스")**·낱개[15].
- 재고 = **박스재고[14]만**(낱개 무시 — 박스로 남아야 올림). **재고금액 = 박스재고 × 박스매입단가[9] = 묶인 돈**, 정렬·스코프 축. 금액 작은 미업로드는 노이즈(올릴 이유 없음).

## 코어 (MVP — 한 모듈: 재고조인 + 환원매칭)
- 입력 자동로드: `product_master.csv` + `listing_<channel>.csv` × 8 (마진모니터 스냅샷 그대로 — 쿠팡 로켓그로스 제외·ESM 지마켓만, 별도 raw 안 만듦). listing 관리코드 컬럼명 = **"코드"**(예 `26-08-02`, 빈값 스킵).
- `resolve_identity(코드, refs) -> [상품코드,...]`: resolve_code 분류 분기 재사용, 매입가/재고 대신 **상품코드** 반환. 합포=구성 관리코드별 상품코드 리스트(다중), 소분=sobun 원코드→상품코드, PC낱개=`코드[2:]`=상품코드, 박스=관리코드→상품코드.
- 의사코드:
```
uploaded[ch] = { sc  for listing[ch]행  for sc in resolve_identity(행["코드"], refs) }
for 상품코드행 in pm_by_prod  (박스[14] > 0):
    재고금액 = 박스[14] × 박스매입단가[9]
    for ch in 8채널:  상품코드 ∈ uploaded[ch] ? "이상없음" : "업로드필요"
품절: 박스재고 ≤ 0 인데 어느 ch uploaded에 존재 → "품절처리필요"
정렬: 재고금액 desc
```

## 페이지 (독립 '업로드감시')
- 표: 관리코드·상품명·박스재고·박스매입가·**재고금액** + 8채널 상태, 재고금액 desc.
- KPI(라이트): **채널별 미업로드 건수만**(대시보드 라이트 유지 — 금액가중/추이 후순위).
- 필터: 채널 선택 → 그 채널 업로드필요만.

## L4 — 탐지 → 등록 핸드오프
- **스마트스토어·ESM**: 선택(또는 재고금액 top N) → `product_input_form_v2.xlsx` prefill → register 워크플로우 인계.
  - prefill 디폴트: 업로드구분=**박스** / 합포장=`product_attributes` 합포수량 **있으면**(신규는 보통 공백) / 과세=**과세 디폴트**(면세는 등록 채팅방 진행 중 사용자 수정).
  - 상품명·카테고리는 register AI 단계가 처리(여기서 안 풂).
- **나머지 6채널**(식봄·캐시노트·배민·쿠팡·올웨이즈·알리): **CSV 리스트만**(자동폼 나중 — register 워크플로우 없음).

## MVP 제외 (나중)
- L5 채널확장(11번가·셀러허브·자사몰·토마토 — 골든 12 vs 모니터 8). listing 파서 추가 필요.
- 추이/스냅샷 저장(Community Cloud 스케줄러 없음 → 후순위).
- 6채널 자동 등록폼.
- 재고금액 × 예상마진 우선순위(baseline_margin 연계).
- 품절 채널 판매상태 정밀화(현재 product_master 재고 단일판정).

## 의존 / 공유 (manifest)
- ← `product_master`(박스재고·박스매입가·상품코드/관리코드·상품명) · `listing_<channel>.csv` ×8(마진모니터 소유, 읽기 공유) · `sobun`(소분 환원) · `product_attributes`(합포수량 = L4 prefill).
- `resolve_code`(채널마진모니터 core) 분류 로직 재사용 → `resolve_identity` 래퍼.
- → register(`product_input_form_v2.xlsx`) 인계.

## 검증 (구현 시)
- `resolve_identity`: 박스/낱개/소분/합포 4유형 환원 → 상품코드 일치(합포 다중반환).
- 골든 ★재고관리시트 대조: 골든 4분기 판정 ↔ 코드기반 결과. 이름기반보다 정확하므로 일부 의도적 차이 가능(이름변형/소분 누락분은 코드기반이 더 잡음).
- **미구현 — 코어부터.**

## 코어 구현 (2026-06-12 — core/workflows/upload_monitor.py)
- `resolve_identity(코드,refs)->[상품코드]`(resolve_code 분류 분기 재사용·상품코드 반환, 합포 다중) + `build_uploaded_sets` + `build_gap_table`(재고금액 desc·노이즈/비판매 제외) + `channel_summary`(채널별 미업로드/품절 건수) + `gap_list_for_channel`. channel_margin_monitor `_nfc·_num·load_references` import.
- **비판매 제외**: 중분류 `반품`·`파렛트`(EXCLUDE_MIDCAT) + 사용자 유지 `reference/upload_monitor_exclude.csv`(상품코드, 없으면 no-op). 부자재(포장박스 '3번'·'16호뚱캔'·랩·테이프·'마트킹 환불 상계액')는 창고존 분류(예 통조림-C동)에 섞여 카테고리로 못 거름 → 발견 시 제외목록 추가. 관리코드 공백도 깨끗한 신호 아님.
- 검증(실데이터): 4유형 환원 OK(합포 `31-03-08-CB-31-03-01`→[003612,002099]). 테이블 1,001행(반품·파렛트 19 제외). 커버리지 예) 스마트스토어 업로드된 567·업로드필요362, 쿠팡 317(로켓그로스 제외)·알리 285. 골든 직접대조 불가(외부참조 수식·결과페이지 캐시 없음·시점차)→로직 레벨.
- **페이지**(`app/pages/7_업로드감시.py`, 배포 2026-06-12): KPI(감시대상·업로드필요/품절 합집합)+채널별 건수표+필터(채널·상태·검색)+8채널 매트릭스 dataframe(재고금액 desc·multi 선택·채널키→라벨)+CSV(선택/화면, 상품코드 포함=제외목록 유지용). 네비 마지막 그룹 등록(📦). 마진모니터 컨벤션 재사용(채널1개 선택형 vs 매트릭스). **core 신규 모듈 → Reboot app 1회.** 다음 = L4 + 제외목록 편집 UI.
  - **필터 UX(2026-06-12, page-only)**: 채널 **체크박스**(체크된 채널만 상태 컬럼 노출·재고금액 옆) + **전체 선택/해제 토글** + **컬럼별 상태필터**(선택 채널마다 selectbox, 행필터=AND. 예 스마트스토어=업로드필요 ∧ ESM=이상없음). 구 채널/상태 selectbox 제거.
