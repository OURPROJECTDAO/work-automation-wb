# easyadmin-register (이지어드민 상품등록) — 플랫폼 인스턴스

> 공통 개념(낱개/박스 · 정제상품명 · 서식 일치)은 **workflows/product-registration-common.md** 참조.
> 이 문서는 **이지어드민 전용**. 신규 상품은 모든 판매채널 외에 이지어드민에도 등록해야 함.

## 채널 성격
- **판매 플랫폼이 아님 — 정산 관련 채널(ERP/정산).** 모든 신규 상품이 거쳐가야 하는 필수 경유 채널.
- 그래서 스마트스토어와 달리 **판매가 공식·카테고리·이미지·수수료 무관**. 명칭·공급처·합포 정보만 입력.

## 양식 / 기입 대상
- 업로드 파일 = `상품등록(실제 업로드파일)` 시트. **69열**. 2행에 `기입` 표시된 **7개 열만** 채움, 나머지 빈칸 유지. 서식 100% 보존(openpyxl read_only 금지).
- 시트 구성: `상품등록(실제 업로드파일)`(양식, 1행 헤더/2행 기입표시) · `예시참고`(등록 후 **export 포맷** — 열 배치 다름, 헤더명으로 매핑) · `넣어야할것`(이번 신규 = 스마트스토어 등록 품목 = 판매자상품코드+정제상품명).
- 데이터행은 **2행부터** (기입표시행 덮어씀).

## 기입 7열 규칙 (낱개 vs 박스)
모드 판별: 판매자상품코드 `PC…` = **낱개** / 그 외(관리코드 `01-09-01` 등) = **박스**.

| 열 | 항목 | 낱개(PC) | 박스(관리코드) |
|---|---|---|---|
| A | 상품명 | `[PC코드][낱개1개][{이름}ㅁ1개입ㅁ]` (레거시) | `[관리코드]{정제상품명}` |
| B | 공급처코드/공급처명 | `20010` (일괄 고정) | `20010` |
| C | 공급처 상품명 | A에서 맨앞 `[PC코드]` 제거 | A에서 맨앞 `[관리코드]` 제거(=정제명) |
| D | 공급처 옵션 | = A (그대로) | `[관리코드]-{정제풀네임} [{용량}*{박스내품}]` |
| E | 원산지 | = C | = C |
| AB | 합포불가 | `4` (고정) | `4` |
| AC | 동일상품 합포가능 수량 | `박스내품` | `합포장`(기본 1) |

### 핵심 포인트
- **낱개 A 레거시 형식**: `{이름}` = 정제상품명에서 끝 ` 1개` 제거. 구분자 `ㅁ` = 한글 자음 **ㅁ (U+3141)** 그대로 박음. `[낱개1개]`·`1개입` 의 1은 낱개 고정(항상 1).
- **박스 D 공급처 옵션**: 풀네임 전체가 들어감(브랜드 분리·축약 안 함). `[규격]` = product_master 규격을 `[용량*박스내품]` 대괄호로(예 `(20g*45)`→`[20g*45]`, `800g*10`→`[800g*10]`). 낱개 D는 A와 동일.
- **AC = "한 번 배송에 묶이는 동일상품 수량"**: 낱개=박스내품(1박스분), 박스=합포장(묶음 박스 수, 기본 1). 사용자가 따로 말 없으면 이 기본값이 정답. 가벼워 여러 박스 묶는 품목만 합포장 별도 지정.
- B=20010(공급처 "기타" 코드)·AB=4 는 전 품목 고정.

## 데이터 출처
- product_master(app repo reference/product_master.csv): 관리코드/상품코드 → 박스내품·규격.
  - 낱개 PC코드 → 상품코드 = `PC` 떼고 앞 0 제거(PC005830→5830)로 조회.
- 정제상품명 = 스마트스토어 등록명(넣어야할것 시트 제공). 낱개 ` 1개` / 박스 ` {박스내품}개` suffix.

## 생성 절차
1. 양식 원본 load_workbook(read_only 금지) → 3시트·69열·서식 보존.
2. `상품등록` 시트 2행부터 7열만 기입(값만 set, number_format 건드리지 않음).
3. 재로드 검증: 헤더/열수(69)/행수/샘플 행 타입 확인.
