# smartstore-register (스마트스토어 상품 등록) — 플랫폼 인스턴스

> 공통 개념(낱개/박스 · 합포장→배송비N · 마진 산정공식 · 상품명 정제 · 카테고리 매핑 · 캐시 · 서식 일치)은
> **workflows/product-registration-common.md** 참조. 이 문서는 **스마트스토어 전용 값**만.

## 표준 입력시트 (사용자 제공 — 2026-06-12 확정)
일상 운영은 **10열 입력시트** 1장으로 받는다(이미지URL 사전기입형, 멀티채널 v2폼 아님). 컬럼:
`상품코드 · 관리코드 · 상품명(raw) · 박스/낱개 · 최대합포(박스기준) · 목표마진율 · 면세/과세 · 카테고리(빈칸=AI채움) · 대표이미지URL · 상세이미지URL`.
- 매입가·박스내품은 시트에 없음 → **product_master.csv 관리코드 조인**(낱개=매입단가/박스=박스매입단가, 박스내품).
- 처리 순서: master 조인 → 판매가/N/배송코드/A코드 산정 → 카테고리 AI매칭(애매=사용자 상의) → 상품명 AI정제(다브랜드/내부표기 정리, 사용자 확정) → 양식 작성·서식검증 → outputs 저장. (logs/2026-06-12-smartstore-batch-12)

## 스마트스토어 전용 값
- **수수료율**: 6%.
- **배송 경제**: 공통(고객 3000 / 실비 2500).

## 양식(일괄등록) — 채우는 13개 컬럼 + 셀 타입/서식
> 양식 93열·헤더 2행(1=섹션, 2=필드명)·예시 2행(3·4, 삭제 후 3행부터 작성). 아래 13개만 채우고 나머지 빈칸 유지. 셀 타입/서식 100% 일치.

| 컬럼 | 항목 | 값 | 타입/서식 |
|---|---|---|---|
| A | 판매자상품코드 | 낱개=PC+상품코드6 / 박스=관리코드 | str / General |
| B | 카테고리코드 | 매핑 | **str**(숫자처럼 보여도) / General |
| C | 상품명 | 정제 | str / General |
| D | 상품상태 | 신상품(고정) | str |
| E | 판매가 | 공식 | **int** |
| F | 단위가격 사용여부 | N(고정) | str |
| J | 부가세 | **'과세상품' / '면세상품'**(실물값) | str |
| L | 재고수량 | 9999(고정) | int |
| W | 대표이미지 | URL | str |
| Y | 상세설명 | `<img src='...' />` | str |
| AD | 원산지코드 | 03(고정) | str + 서식 `@`(앞자리0 보존) |
| AI | 배송비템플릿코드 | N별 매핑 | **int** |
| BD | A/S템플릿코드 | 3379093(고정) | int |

## 고정값 (스마트스토어 양식)
- 상품상태=신상품 · 단위가격여부=N · **부가세='과세상품'**(면세는 '면세상품') · 재고=9999 · 원산지='03'(@텍스트) · A/S=3379093.
- ⚠️ J는 '과세'/'면세'가 아니라 **'과세상품'/'면세상품'** (양식 실물 대조, 2026-06-12).
