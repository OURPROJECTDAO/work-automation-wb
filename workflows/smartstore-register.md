# smartstore-register (스마트스토어 상품 등록) — 멀티채널 상품 등록

> 운영 방식: **챗 네이티브 + KB** (풀 Streamlit 템플릿화 X). decisions/0009.
> 현재 스마트스토어만. 약 7~8개 채널로 확대 예정 (채널별 config·reference만 추가).

## 한 줄 요약
상품관리(product_master) → 채널별 일괄등록 양식(.xlsx) 변환. 결정적(코드/가격/배송비/이미지/서식)은 코드, 판단(상품명 정제·카테고리 매핑)은 Claude(AI+웹검색). **셀 타입/서식까지 양식과 100% 일치 필수.**

## 아키텍처 (왜 챗 네이티브인가)
- 결정적 90%: 모드분기·판매자상품코드·판매가공식·합포→배송비코드·이미지URL·고정값·서식맞춤 → 코드.
- 판단 10%: 상품명 정제(판매용 가공)·카테고리 매핑(애매 시 웹검색) → Claude.
- 풀 Streamlit이면 판단 10%를 사람이 수기로 해야 함(핵심 미자동화). 챗은 둘 다 처리. 기존 아키텍처(Streamlit=무LLM 생산 / Claude=판단)와 일관.
- 향후: 결정적 코어 = 채널 config로 구동되는 재사용 엔진 / 관리코드→{정제상품명,카테고리코드} 캐시 축적 → 캐시 히트 시 완전 결정적 → 충분히 차면 재업로드만 Streamlit/Claude API로 분리 가능(지금 미구현).

## 두 업로드 모드
| | 낱개(PC) | 박스 |
|---|---|---|
| 판매자상품코드(A) | `PC`+상품코드(6자리 zero-pad) 예 PC005830 | 관리코드 그대로 예 01-09-01 |
| 매입가 | 매입단가(낱개) | 박스매입단가 |
| 규격 suffix(상품명 끝) | `1개` | `{박스내품}개` 예 45개 |
| 합포 배송비 N | 합포장 × 박스내품 | 합포장(박스 개수) |
- 모드는 소스 시트의 업로드 컬럼(박스/낱개)으로 사용자가 지정. 저가·소용량은 보통 박스(낱개로 올리면 판매가가 비현실적으로 낮아짐).

## 판매가 공식 (실물 검증 완료 — 글로벌하베스트)
```
정산액 = (판매가 + 배송수취) × (1 − 수수료)
마진율 = (정산액 − 매입가 − 실택배비) / 정산액
→ 판매가 = ⌊ (매입가 + 실택배비) / [(1−수수료)(1−마진율)] − 배송수취 ⌋ (100원 내림)
```
- 수수료는 (판매가+배송비) **합계**에 부과. 마진 분모 = 정산액.
- 스마트스토어 기본값: 수수료 6%, 배송수취 3000, 실택배비 2500.
- 마진율 = **매 배치 사용자 입력**(디폴트 10%).
- 검증: 매입 2915·마진10% → 3400 / 매입 2850 → 3300 (양식 예시와 원 단위 일치). ※ 시트의 매출단가/낱개매출단가는 의미 없음(공식으로 산정).

## 상품명 정제 규칙 (Claude 제안 → 사용자 확정 → 캐시)
- 붙은 단어 띄어쓰기(갈릭새우칩→갈릭 새우칩), 슬래시 제거(푸드킹/찹쌀누룽지→푸드킹 찹쌀누룽지).
- 내부용 괄호/수량 제거((72),(42),/128,*20,*10 등).
- 브랜드 앞으로(명가·푸드킹·동서·라인프렌즈).
- 용량 원본에 있으면 유지, 끝에 모드별 suffix(낱개 `1개` / 박스 `N개`).
- 상품 수 많으면 AI가 일괄 후보 제안.

## 합포장 → 배송비템플릿
- 배송비템플릿 = "**N개당 3000원**" 규칙.
- N: 박스모드 = 합포장(박스 수) / 낱개모드 = 합포장 × 박스내품.
- 합포장 = 소스 시트 컬럼(1박스 기준, 합포 가능 박스 수). 예: 코카콜라 24개/박스, 합포 2박스 → 박스로 2개당 3000.
- N → 배송비템플릿코드 (**스마트스토어 전용**, 채널별 다름·사용자 제공):

| 1 | 2745710 | 8 | 2856015 (⚠중복 2756602) | 24 | 3241202 |
| 2 | 2745660 | 10 | 2856017 | 30 | 3377508 |

- 보유 N 외 값(7·9·11·13·14·…) 필요 시 템플릿 추가해야 함. 특히 박스내품 14를 낱개로 올리면 N14 없음.

## 카테고리 매핑 (Claude: AI + 웹검색 → 캐시)
- 룩업표: reference/smartstore_category_food.csv (630개; 카테고리번호·대분류·중분류·소분류·세분류, 전부 식품).
- 채널마다 카테고리 양식 다름 → 사용자가 채널별 표 제공.
- 상품이 어느 카테고리인지 애매하면 웹검색으로 제품 성격 확인 후 매핑 → 관리코드→카테고리코드 캐시.
- 예(글로벌하베스트): 새우칩·메밀칩·누룽지과자 → 일반스낵(50022500) · 그린티 → 녹차/홍차음료(50012001) · 우롱티·자스민티 → 기타전통/차음료(50012002) · 요구르트스파클링 → 기타탄산음료(50002257) · 동서커피믹스 → 커피믹스/인스턴트커피(50002606).

## 이미지 URL
- 대표이미지(W): `https://gi.esmplus.com/td680708/{관리코드}/{관리코드}_A1.jpg`
- 상세설명(Y): `<img src='https://gi.esmplus.com/td680708/{관리코드}/{관리코드}_B1.jpg' />`
- 확장자(jpg/png) **상품별로 다를 수 있어 URL 실검사 필요**. gi.esmplus.com이 샌드박스 네트워크 차단(host_not_allowed) → 검증하려면 설정>Capabilities>Code execution 도메인 허용목록에 gi.esmplus.com 추가 + **새 세션**에서 검사. (글로벌하베스트 배치는 전부 jpg 확인.)

## 양식(일괄등록) 스펙 — 채워야 하는 13개 컬럼 + 셀 타입/서식
> ⚠️ 셀 타입/서식(일반/텍스트/숫자)까지 양식과 100% 일치. 생성 = **양식 원본 복사 → 예시행 삭제 → 데이터행 작성(컬럼별 number_format 그대로 복사)**. openpyxl read_only 금지.

| 컬럼 | 항목 | 값 | 타입/서식 |
|---|---|---|---|
| A | 판매자상품코드 | PC+상품코드 / 관리코드 | str / General |
| B | 카테고리코드 | 매핑 | **str**(숫자처럼 보여도) / General |
| C | 상품명 | 정제 | str / General |
| D | 상품상태 | 신상품(고정) | str |
| E | 판매가 | 공식 | **int** |
| F | 단위가격 사용여부 | N(고정) | str |
| J | 부가세 | 과세/면세(**사용자 입력**) | str |
| L | 재고수량 | 9999(고정) | int |
| W | 대표이미지 | URL | str |
| Y | 상세설명 | img 태그 | str |
| AD | 원산지코드 | 03(고정) | str + 서식 `@`(앞자리0 보존) |
| AI | 배송비템플릿코드 | N별 매핑 | **int** |
| BD | A/S템플릿코드 | 3379093(고정) | int |
- 양식은 93열. 위 13개 외 빈칸 컬럼은 비워둠. (사용자 규칙: 채워진 칸=필수, 빈칸=불필요)

## 고정값 / 사용자입력 / AI 구분
- 고정: 상품상태=신상품, 단위가격여부=N, 재고=9999, 원산지=03, A/S=3379093.
- 사용자입력(매 배치): 마진율, 모드(박스/낱개), 합포장, **과세/면세**, 이미지확장자.
- AI(+사용자 확정): 상품명 정제, 카테고리코드.
- 채널별 reference: 양식·컬럼맵·카테고리표·배송비 N→코드맵·수수료율·가격파라미터(배송수취/실택배비)·고정값·캐시.
