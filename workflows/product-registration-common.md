# product-registration-common — 상품 등록 공통 개념 (전 채널 공유)

> 모든 판매 채널(스마트스토어·쿠팡·ESM·…)의 상품 등록에 공통 적용되는 개념·규칙.
> 채널별 구체값(양식·카테고리코드·배송비코드·수수료)은 각 workflows/<channel>-register.md.
> 운영 방식: 챗 네이티브 + KB (decisions/0009). 약 7~8개 채널 확대 예정.

## 공통 개념 (채널 무관)

### 1. 낱개 / 박스 모드
- 한 상품을 **낱개(최소단위)** 또는 **박스(묶음)** 로 올림. 소스 시트 업로드 컬럼으로 품목별 사용자 지정.
- 판매자상품코드 (우리 관리코드 기반):
  - 낱개 = `PC` + 상품코드(6자리 zero-pad). 예 PC005830.
  - 박스 = 관리코드 그대로. 예 01-09-01.
- 매입가: 낱개 = 매입단가(낱개), 박스 = 박스매입단가.
- 상품명 끝 규격: 낱개 `1개`, 박스 `{박스내품}개`.
- 저가·소용량은 보통 박스(낱개로 올리면 판매가가 비현실적으로 낮아짐).

### 2. 합포장 → 배송비
- 배송비는 "**N개당 3000원**" 규칙으로 매김.
- 합포장 = 1박스 기준, 묶음배송 가능한 박스 수 (소스 시트 컬럼).
- N = 박스모드: 합포장 / 낱개모드: 합포장 × 박스내품.
- 예: 코카콜라 24개/박스, 합포 2박스 → 박스로 2개당 3000 / 낱개로 48개당 3000.
- 채널에는 이 N에 해당하는 "배송비템플릿코드"를 넣음 (코드 값은 **채널별**).

### 3. 마진율 / 판매가 산정 방식
```
정산액 = (판매가 + 배송수취) × (1 − 수수료율)
마진율 = (정산액 − 매입가 − 실택배비) / 정산액
→ 판매가 = ⌊ (매입가 + 실택배비) / [(1−수수료율)(1−마진율)] − 배송수취 ⌋ (100원 내림)
```
- 수수료는 (판매가+배송비) **합계**에 부과, 마진 분모 = 정산액.
- 배송 경제(공통): 고객 배송수취 3000 / 우리 실택배비 2500.
- 마진율 = **매 배치 사용자 입력**(디폴트 10%).
- 수수료율은 **채널별**(예: 스마트스토어 6%).
- 시트의 매출단가/낱개매출단가는 의미 없음(공식으로 산정).

### 4. 상품명 정제 (Claude 제안 → 사용자 확정 → 캐시)
- 붙은 단어 띄어쓰기 · 슬래시 제거 · 내부용 괄호/수량((72)·/128·*20 등) 제거 · 브랜드 앞으로 · 용량 유지 + 모드 suffix.
- 상품 수 많으면 AI 일괄 후보 제안.

### 5. 카테고리 매핑 (Claude: AI + 웹검색 → 캐시)
- 채널별 카테고리 룩업표에서 상품 → 카테고리코드.
- 애매하면 웹검색으로 제품 성격 확인 후 매핑.

### 6. 캐시 전략 (미구현)
- 관리코드 → {정제상품명, 카테고리코드} 를 **채널별로** 축적.
- 신규 상품만 AI/검색, 재등록은 결정적 → 카탈로그 안정 시 자동화율↑.

### 7. 양식 서식 일치 (공통 원칙)
- 출력 .xlsx는 채널 양식의 셀 타입/서식(일반/텍스트/숫자)까지 **100% 일치**.
- 생성 = 양식 원본 복사 → 예시행 삭제 → 데이터행 작성(컬럼별 number_format 그대로 복사). openpyxl read_only 금지.

## 플랫폼별로 달라지는 부분 (각 channel 문서에 기록)
- 일괄등록 **양식**(컬럼 레이아웃) + **어느 컬럼을 채우는지**(컬럼맵)
- **카테고리코드 표** (채널별 분류 체계)
- **배송비템플릿 N→코드 맵**
- **수수료율**
- **고정값** (상품상태·원산지·A/S템플릿·단위가격여부·부가세 기본 등 — 양식 컬럼에 종속)
- **이미지 넣는 컬럼/형식** (이미지 호스트·파일 패턴은 우리 공통: `gi.esmplus.com/td680708/{관리코드}/{관리코드}_A1|B1.{확장자}`, **확장자=jpg→png URL 실검사 자동판별**, 사용자 입력 아님)

## 사용자입력 / AI / 고정 구분
- 사용자입력(매 배치): **대상 채널** · 마진율 · 모드(박스/낱개) · 합포장 · 과세/면세. (합포장·과세는 스마트스토어용; G마켓만이면 무시)
- AI(+사용자 확정): 상품명 정제 · 카테고리코드.
- 자동(입력 불필요): 이미지 확장자 — 관리코드 URL(jpg→png) 실검사로 판별. (gi.esmplus.com 접근 가능)
- 고정/채널 reference: 위 "플랫폼별" 슬롯.
