# 워크플로우: intelligence-layer (지능 레이어 — 이력 엔진 + 두뇌: 마진·재고·A/B)

> 이 워크플로우를 건드리기 전 이 파일을 읽는다. 전역 함정 pitfalls.md, 설계근거 decisions/0018, 공유자산 manifest.md.
> ⚠️ **미구현 — 설계확정.** 직접 실행(코드/적재)은 다음 세션. **이 문서가 인계 정본.** (2026-06-12 설계)

## 0. 한 줄
관찰(대시보드·마진모니터·업로드감시)과 실행(가격변경·등록·발주·송장)은 **이미 다 있고**, 그 사이 '두뇌'(진단·추천·측정)가 빈칸. 매일/3년치 데이터를 private repo에 **이력으로 쌓아(이력 엔진)** → ① 마진 침식·제시 ② 입고·품절 예측 ③ 채널 가격 A/B 를 돌린다. 데이터는 부족하지 않다 — 안 모으고 있었을 뿐.

## 1. 큰 그림 (2층 + 루프)
- **루프**: 관찰 → 진단 → 추천 → 실행 → 측정 → (학습) → 관찰. 관찰·실행은 구축됨, 진단·추천·측정 = 두뇌(빈칸).
- **Layer 1 이력 토대(메모리)**: 과거 소급 4종 + 현재 앵커 1종 → private repo(`work-automation-data`) 날짜 파티션. 대시보드 parquet 패턴 재사용.
- **Layer 2 두뇌(결정)**: ①마진 침식·제시 ②입고·품절 예측 ③채널 가격 A/B.
- **실행 = 기존 actuator 재사용**: 가격변경 7채널(channel-margin-monitor)·상품등록·발주(logistics)·송장(invoice-fill).

## 2. ★중요 사실 / 정정 (헷갈리면 안 됨 — 사용자 명시)
- **(A) 정산 진실원천 = 매출자료** (천년경영 매출 업로드 정본 = 대시보드 영업이익현황 소스). 판매금액=정산. EasyAdmin·erp(천년경영업로드 output) 정산은 **raw 상류**(매일 추가정제 대상) → 진실 아님. "더 정확할 필요 없음". 따라서 채널별 정산내역서를 따로 파싱할 필요 없음.
- **(B) 택배 실배분 = 송장번호 그룹** (EasyAdmin). 같은 송장번호 = 한 박스(합포 구성). 대시보드 온라인 마진 탭의 **추정송장**(수량÷(합포수량×박스내품))·보정계수 k 를 **실제값으로 대체** → 상품별 마진이 추정→정확. 송장번호는 EasyAdmin 주문 export([10])에도 있어 별도 송장출력 파일 불요(백필도 커버). **✅ 구현(2026-06-15, ship_alloc.py): rate=EA박스÷매출낱개(★매출=낱개·EA=판매단위, 배율≈박스내품)·합포는 물류량 가중·(상호명,월) 00-12 정합 스케일. 매핑 ESM=G마켓+옥션·배민=상회+대용량·자사몰 제외·EA 미경유 6채널 추정 fallback.**
- **(C) 매입가 두 출처**: master 매입단가(수정로그가 변경 기록) vs 유형별매입현황 실입고 단가. 마진모니터/실현마진은 **master 기준**. 차이 자체가 신호(매입현황↑인데 master 미수정 → 장부마진 과대, 곧 수정 시 마진 급락 = 조기경보). **✅ 실증(2026-06-15, logs/2026-06-15-2c-validation): 자동 연동 아님 — 독립·수동(반자동).** 운영자가 입고 단가 보고 master를 손수정. 근거=수정후 값이 ±7일 실입고가와 일치 35%뿐(자동이면 ~100%)·시차 당일41%·≤7일77%·>30일8%·오타정정 흔적(4000→5000→476→4762). → **2C 시차 윈도우 실재(>7일 미반영 23%) = 조기경보 성립 확증.**
- **(D) 마진모니터 정산식**(스마트 판매가×0.94 등)은 *앞으로 가격 정할 때*(미판매 리스팅) **추정용**. *실현* 마진 = 매출자료(정산) + 수정로그(매입가) + 송장(택배 실배분).
- **(E) Community Cloud 스케줄러 없음** → 알림은 GitHub Actions cron(repo 커밋 자료 diff→webhook) 또는 인앱 스냅샷 diff(지난 스냅샷 대비 입고/품절). 추이 저장 후순위였던 이유.

- **EasyAdmin 주문 ≠ 매출자료 100% 정합 안 됨(사용자 명시)** — ① 발주 후 ERP 일괄입력 timing ② 취소돼 안 나간 송장을 채널 00-12에서 미차감(송장 과대). EasyAdmin=velocity·송장그룹 전용, 정산 진실=매출자료(§2A). **일별 velocity 덩어리짐 → 월/주 단위 사용.** P2 송장배분 시 phantom 송장은 매출 조인에서 탈락 처리.
