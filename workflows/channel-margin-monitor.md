# 워크플로우: channel-margin-monitor (채널 가격·마진 모니터)

> 이 워크플로우를 건드리기 전 이 파일을 읽는다. 전역 함정은 pitfalls.md, 공유자산 연결은 manifest.md.

## 요약
- 판매 채널의 라이브 리스팅(상품관리 다운로드)을 받아 상품별 마진율을 계산 → 기준마진 대비 이탈 탐지 + 기준마진 달성 권장가를 역산하는 모니터.
- 등록(신규)이 아니라 **운영 중 리스팅 점검**. 사용자 엑셀([1]마스터 수식 조인본) 대체.
- 스마트스토어 먼저. 타 채널은 서서히 추가(baseline_margin이 이미 10채널 보유).
- 상태: **진행중** — 코어+페이지 구현 완료(2026-06-10). 첫 배포 Reboot·실사용 확인 대기.

## 입력 (매 실행 1 업로드)
- 채널 상품관리 다운로드. 스마트스토어 = `[일괄수정 (전체 업로드)]` .xlsx.
  - r2=헤더, r3~5=가이드행(업로드시 삭제 안내문), r6+=데이터.
  - 사용 컬럼: A 상품번호(키) · B 판매자상품코드(관리코드/PC/변환코드) · D 상품명 · F 판매가 · AO(41) 기본배송비 · BF(58) 즉시할인 · BQ(69) 포인트지급 · BZ(78) 판매자바코드(N).

## 마진 계산 (확정 공식)
```
N        = 판매자바코드 (빈값/0 → 1; 그 관리코드 기준 판매수량 배수. PC=낱개1개 base)
매입가    = (코드해석 base) × N
판매가net = 판매가 − 즉시할인 − 포인트                 (다운 F − BF − BQ)
정산액    = 판매가net × 0.94 + 배송비 × 0.967          (0.94=수수료6%, 0.967=배송비 별도정산, 배송비=AO)
이익      = 정산액 − 매입가 − 2700                     (실택배비 2700 단일)
마진율    = 이익 / 정산액
탐지      = 마진율 − 확정마진율
권장가    = ⌊ ((매입가 + 2700)/(1−확정마진율) − 배송비×0.967) / 0.94 ⌋   (100원 내림)
```
- 우리 등록 공식(이익/정산액 분모)과 동일. **단 실택배비 = 2700 단일**(엑셀 원본 3000/3700 폐기 — 합포장 묶음판매 +700 접음, 사용자 확인 2026-06-10).
- 수수료율·확정마진율은 채널별. 스마트스토어 수수료 6%.

## 코드 해석 (매입가/재고/규격) — 4-tier
| 코드 형태 | base 매입가 | 재고 | 규격 | 소스 |
|---|---|---|---|---|
| 박스 (`24-07`) | 박스매입단가[9] | 박스[14] | 규격[6] | product_master (관리코드[4]) |
| PC낱개 (`PC`+상품코드6자리) | 매입단가[8] | 낱개[15] | 규격[6] | product_master (상품코드[3], PC 제거) |
| 소분 (`변환코드-원코드`) | 원박스매입단가 ÷ 내품나누기 | 원코드 박스재고 | 소분규격 | sobun.csv(변환관리코드→원코드·내품나누기) + product_master |
| 합포 (`코드1-CB-코드2[-CB-코드3]`) | Σ 박스매입단가 + 700 | Σ 박스재고 | 다운로드/합포규격 | -CB- 코드 파싱 + product_master (reference 없음) |
- 최종 매입가 = base × N(판매자바코드).
- **판매자바코드 N(수량배수)과 합포 -CB-(구성)는 직교** — 둘 다 적용.
- 소분 = PC 이전 레거시(내품나누기 임의값 → 코드 역산 불가, reference 필수). 추후 PC 낱개로 교체 예정, 현재 잔존.
- 합포 = 시스템 통합(코드 파싱+700).

## Reference (app `reference/`, 신규 2026-06-10)
- `baseline_margin.csv` (1,229): 관리코드 + 10채널 확정마진율(스마트스토어·자사몰·ESM·배민상회·식봄·올웨이즈·캐시노트·쿠팡·토마토·알리). 채널 열만 골라 씀. 키에 박스/PC/소분/합포 혼재(판매자상품코드 직조인). '+' 코드 보존. **다중채널 공유**(manifest A).
- `margin_floor.csv` (61): 관리코드·상품명·비고·제한내용(텍스트). 제조사 단가하한. 숫자 클램프 X — 권장가 칸에 텍스트 표시(휴먼판단). 전 채널 적용(별도 지시 전까지).
- `sobun.csv` (136): 변환관리코드·원코드·내품나누기·소분규격·소분상품명. 매입가/재고 런타임 산출.
- 연동: `product_master.csv`(매입가·재고·규격·박스내품 — 다중소비자, 매일 갱신).

## 전용 함정
- 다운로드 r3~5 가이드행 skip(r2 헤더, r6+ 데이터). 판매자상품코드 빈 행 존재(4/712).
- **판매자바코드(BZ)**: "판매단위가 관리코드 기준 몇 배인가" — 플랫폼에 영속(다운로드마다 리스트 변동 → 별도 매핑표 회피). 빈값→1. **매입가×N 반드시 반영**(빼면 N>1 리스팅 마진 과대).
- 마진제한은 마진율 계산엔 무관 — 권장가 표시만 텍스트로 대체.
- 미해결: baseline↔product_master 조인 갭(미설정/미등록), 마진제한 적용채널 세분화, sobun↔unit_list↔sub_list 개념중복(통합 추후).
