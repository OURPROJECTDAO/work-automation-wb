# 워크플로우: sikbom-event-planning (식봄 행사기획)

> 이 워크플로우를 건드리기 전 이 파일을 읽는다. 전역 함정은 pitfalls.md, 공유자산 연결은 manifest.md. 정산식·원가해석은 channel-margin-monitor.md 재사용.

## 요약
- 식봄 MD와 함께 기획전(행사) 품목·행사가를 정하는 **분석 업무**(챗 네이티브). 식봄이 주는 "종합 프로모션 리스트" 양식에서 MD 요청(카테고리·타겟단가)에 맞는 **우리 식봄 판매중 상품**을 찾고 → 매입단가·재고·마진을 계산 → 행사가를 제안.
- 산출물 = **내부 공유용 표(기획전별 시트)**. 식봄 공식 양식의 우측 '판매자' 칸은 사용자가 직접 기입.
- 본질: "특정 타겟단가에 맞출 품목이 있나 / 맞추면 얼마 남나 / 행사 품목 찾기". 매 회차 뉘앙스 유사.
- 상태: **운영중** (1차 2026-07 기획전 처리).

## 입력
### (1) 식봄 행사 양식 (예: `식봄_7월_기획전_종합_프로모션_리스트.xlsx`)
- **시트 = 기획전 단위**(예: 여름메뉴 / 양식(서양)). 각 시트 동일 레이아웃:
  - 좌측 B:G "식봄 가이드"(MD 요청): B순서·**C카테고리·D상품·E주요 단량/스펙·F요청 타겟판가**·G행사기간. MD가 일부 행만 지정.
  - 우측 H:O "판매자"(우리 기입): H순서·I카테고리·J상품no·K업체명·L상품명·M배송형태·N정가·**O업체 행사가**.
  - Q:U "노출 카테고리" 버킷(행 단위) — MD가 원하는 카테고리 묶음(지정 안 한 빈 버킷도 채울 수 있음).
  - row4 H='ex.' 예시행(면사랑 칡냉면).
- ★ **타겟단가 단위 = MD 스펙 단위**. 탄산 '355ml*24입'=박스가, 골뱅이 '140g/400g'=캔당, 식용유 '18L'=낱개.

### (2) 식봄 판매중 상품 리스트 (식봄 상품관리 다운로드, 예 `상품리스트_*.xlsx`)
- 시트 'Worksheet'. r1~3 유의사항(다운로드 전용), **r4=헤더, r5+=데이터**.
- 컬럼: A상품no(고유키)·**B상품코드(=관리코드)**·C/D/E 1·2·3차 카테고리·F상품명·G판매규격·K제조사·**O매입단가=전부 0(식봄이 가림)**·P정가(가짜)·**S판매단가(=실제 판매가)**·AB판매여부(y=판매중).
- ★ **O매입단가는 0이라 못 씀** → 원가는 product_master에서 조인. ★ **P정가는 가짜가격**, 실판매가는 S(판매단가).
- "현재 식봄에 판매중인 상품 기준" — 이 리스트가 후보 모집단(전건 판매여부 y).

## 식봄 정산식·마진 = channel-margin-monitor 식봄 표준 (불변, 동일)
```
정산액 = 판매가 × (1−0.07) + 3000 × 0.967         (식봄 수수료 7% · 배송비 상수 3000)
이익   = 정산액 − 매입가 − 2700                     (실택배 2700)
마진율 = 이익 / 정산액
하한가(목표마진 M) = ⌈ ((매입가 + 2700)/(1−M) − 3000×0.967) / 0.93 / 100 ⌉ × 100   (100원 올림)
  - M=0 → 손익분기價,  M=식봄 baseline → 기준마진價
```

## 행사가 산정 로직
- **타겟마진** = MD 요청 타겟판가로 팔 때 마진율(위 식, 판매가=타겟).
- **업체 행사가 = min(현재 실판매가 S, MD 타겟)**:
  - 현재가 > 타겟 → **타겟까지 인하**(행사가=타겟). 마진=타겟마진.
  - 현재가 ≤ 타겟 → **현재가 유지**(이미 타겟 이하 → 인상 안 함). 마진=현재마진. (사용자 확정 a안)
  - (옵션 b: 타겟이 현재보다 높을 때 타겟까지 인상 = 마진↑이나 행사인데 인상이라 비표준)
- **마진율 = 행사가 기준**.

## 매칭 함정 (★ 핵심)
- **카테고리 매핑 안 됨**: MD 노출 카테고리(냉면사리·탄산음료·올리브오일 등 행사 버킷)는 식봄 1/2/3차 카테고리와 직접 매핑 X → **상품명 키워드 매칭** 필요.
- **단위 미스매치**: MD 스펙이 낱개(140g·400g·1kg·1L)인데 우리는 **박스만 판매**(48개·24개·6개) → **낱개/캔당 환산**(박스원가÷내품) 후 타겟과 비교. 타겟이 우리 낱개 원가 미만이면 손익분기도 안 됨. (예 골뱅이 140g: 최저 동원 통큰세일 캔당원가 2,700, 손익분기 캔당 2,899 > 타겟 2,815 → 불가. 400g: 동표 을지로 캔당 6,600·손익분기 7,088 < 타겟 7,461 → 가능)
- **올리브오일 vs 올리브(통조림)** 등 동음 오매칭 주의(상품명에 '올리브' 들어가도 식탁김·블랙올리브슬라이스는 오일 아님).
- **미취급/규격 없음**은 제외(예 식용유 18L 없음=1.8L팩만, 치킨스톡·바질 미취급, 스파게티니9번 없음).
- **음수 재고**(초과판매·품절) 주의 — 행사 전 재입고 확인(예 코카콜라제로 박스재고 −152).
- **★ listing 스냅샷 stale 주의 (2차 사례 2026-06-23)**: 작업 전 `reference/listing_sikbom.meta.json` updated_at 확인. 사용자가 "방금 식봄 갱신"이면 무조건 재fetch — 06-18 스냅샷으로 전건 미등재 오판한 사례. 식봄 등재 직후 상품번호·현재 판매가가 새로 잡힘.
- **★ 식봄 등재 유닛(PC낱개 개당 vs 박스)별 단위 일치 (2차 사례)**: listing 코드 tier로 판단 → 매입가·판매가를 **같은 유닛**으로 계산. 같은 관리코드라도 식봄엔 **PC낱개(개당)**로 올릴 수 있음(예 새우칩 240g·컨츄리타임 레몬에이드는 개당, 새우칩 20g/90g은 박스 45개/14개). 박스=박스매입단가[9], PC낱개=낱개매입단가[8]. 유닛 틀리면 마진 전부 오류.
- 업체명 = 제조사(K). PC낱개 제품은 제조사칸 비어 상품명 브랜드 사용(곰표·라폰테·보베다).
- 배송형태: 식봄 양식은 싱싱/택배 등. 상온건은 '택배배송'(다운로드 M '택배배송비명'은 배송비 정책명이라 ≠ 배송형태).

## 산출물 (내부 공유용 xlsx, 시트=기획전별)
- 컬럼: **카테고리·상품no·업체명·상품명·배송형태·정가(식봄=가짜)·판매단가(실판매)·식봄 타겟단가·업체 행사가·매입단가·재고(박스)·마진율·비고**.

- 저마진(<5%) 노랑·음수재고 빨강 강조. 마진율은 하드코드(매입가 컬럼만으로 식봄 정산식 단일셀 수식화 불가).
- 식봄 공식 양식(우측 H:O)은 사용자가 이 표 보고 직접 기입.
