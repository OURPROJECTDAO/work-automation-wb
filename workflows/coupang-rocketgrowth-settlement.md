# 워크플로우: 쿠팡 로켓그로스 정산 (coupang-rocketgrowth-settlement)

> 이 워크플로우를 건드리기 전 이 파일을 읽는다. 전역 함정은 pitfalls.md.

## 요약
- 쿠팡 로켓그로스(RG)는 쿠팡 자체 창고에서 직접 배송 — 우리 쪽 발주(포장·배송) 자체가 없어서
  **cheonnyeon-upload 파이프라인(발주자료 기반)에 애초에 안 잡힘**. 별도 보충 입력 필요.
- 매달 반복 예정. **5월은 사용자가 수작업**(쿠팡로켓데이터집계v1.xlsx)으로 했고, **6월부터 챗 네이티브로 재현**(2026-07-02).
- 산출물 = `쿠팡로켓그로스_YYYY월_천년경영입력용.xlsx` (사용자가 천년경영에 수기 입력).

## 입력 (쿠팡 WING에서 매달 새로 받아야 하는 것)
1. **판매통계 export**(`Statistics-YYYYMMDD_YYYYMMDD.xlsx`, 시트명 '판매통계') — 옵션ID별 순매출·순수량.
   ⚠️ **마지막 행에 'TOTAL' 합계행이 항상 있음 — 반드시 제외하고 합산할 것**(안 그러면 정확히 2배로 잡힘, 2026-07-02 실수 사례).
2. **쿠팡상품관리(바코드,잔여수량) export** — 옵션ID→**업체상품코드(=관리코드)** 직접 매핑. 이게 핵심 소스,
   상품명 fuzzy매칭보다 훨씬 정확. ⚠️ 이 export의 업체상품코드 칸이 **빈 문자열인 행이 있음**(값은 없는데 키는 존재) —
   `if oid in dict` 같은 멤버십 체크만 하면 안 걸러짐, **truthy 체크 필수**(`dict.get(oid)`가 빈 문자열이면 폴백).

## 재사용 가능한 매핑 소스 (달마다 안 바뀌는 부분)
- 전월(또는 그 이전) 매칭본의 **"쿠팡상품관리" 시트**(옵션ID→관리코드) + **"수량매칭" 시트**(옵션ID→수량매칭 배수·기준마진율)는
  옵션ID가 유지되는 한 다음 달에도 그대로 재사용 가능. 매달 재확인 대상은 **신규 옵션ID**(신상품·리스팅 재발급)뿐.
- 매핑 우선순위: ① 쿠팡상품관리 export(관리코드 칸 truthy) → ② 이전 달 "수량매칭" 시트의 코드 → ③ product_master 텍스트 재검색(브랜드+규격) → ④ 그래도 안 되면 사용자에게 가져감.
- **수량매칭**: 옵션명의 "n개, ~g/ml" 패턴 → 대부분 n=실제 낱개수(수량매칭=n 그대로, 박스 배수 아님). 애매하면 **내재단가(순매출÷실제수량)를 product_master 매출단가와 비교**해서 판별(사용자 팁, 2026-07-02).

## 계산식
- **수수료율(로켓그로스) = 11.66%** (일반 쿠팡 마켓플레이스 12%와 다름 — cheonnyeon-upload의 쿠팡전체 H=F×0.88/D와 헷갈리지 말 것).
- 실제수량 = 순 판매 상품 수(옵션 판매횟수) × 수량매칭
- 정산금액 = 순매출 × (1 − 0.1166)
- 실제기입단가(H, 천년경영용) = 정산금액 ÷ 실제수량
- 택배비(G) = 로켓그로스는 쿠팡이 배송 부담 → 0/해당없음.

## 코드 / 처리 방식
- **챗 네이티브(앱 미탑재)**. 매달: (1) 판매통계 export 받기 (2) 전월 매칭본의 쿠팡상품관리·수량매칭 시트로 옵션ID 매핑 시도
  (3) 안 되는 것만 product_master 재검색 → 그래도 안 되면 사용자에게 리스트로 가져감 (4) 관리코드별 집계 → 산출 xlsx.
- 산출 xlsx 3시트: 원본상세(옵션별, 감사추적용) · 천년경영_입력용(관리코드별 집계, 수수료율 B2 셀 조정 가능한 수식) · 미해결(매출0·참고).
- 재사용 파이썬 로직은 이번 세션 ad-hoc(코드 파일로 박제 안 함, sikbom_price_lookup.py 같은 재사용 스크립트는 다음에 반복 패턴 확정되면 고려).

## 검증 (2026-07-02, 첫 실행)
- 6월(2026-06-01~06-30) 106개 옵션 중 **103개 매핑**(84개 기존+5개 재검색+6개 사용자 확인+8개 수량매칭 시트 폴백 버그수정으로 추가 해결).
  나머지 3개는 전부 매출 0(화남 황도 슬라이스·펭귄 양념꽁치·미드봉 흑후추 2개) — 안 급함.
- 매핑 후 순매출 합계가 원본 통계파일 TOTAL행 제외 합계(64,000,650원)와 정확히 일치 — 매핑 커버리지 100%(매출 기준) 확인 완료.
- 정산금액(11.66% 차감) = 56,538,174원, 실제수량 8,026개, 관리코드 31개.
