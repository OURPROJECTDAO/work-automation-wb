# 2026-06-16 두뇌① 탭D 택배 배분 — 합포 2시나리오 정확화 (시나리오1 버그 + 시나리오2 물리합포)

## 무엇 / 왜
탭D 당일 점검은 easyadmin_*.parquet(월·2주 지연)을 못 쓰는 당일 신호라, 매일 갱신 3파일(product_master·송장출력·천년경영업로드 output)로 채널·상품 정밀 마진을 계산한다. 그 정밀도의 최대 변수가 **택배비**인데, 합포(여러 품목/박스가 한 택배로 묶임) 미반영으로 저마진 캔류에 **거짓 역마진**이 났다(0615 정정 후에도 잔존 우려, 사용자 제기). 합포 2종을 모델에 넣어 택배를 실제 물리 박스에 맞춤.

## 합포 2시나리오 (사용자 정의)
- **시나리오1 — 250/355 자동 합포분리**: 이지어드민이 H열 한 행에 콤마로 2품목을 묶어 보냄(예 `250ml 캔음료수-[[31-01-04]코카245] 1개, 250ml 캔음료수-[[31-13]파워에이드240] 1개`). 송장 1건=박스 1, 그 안 물류량 분할.
  - **★버그 발견**: 구 `_resolve_code`가 H열에서 첫 유효코드(코카245)에서 즉시 return → **둘째 품목(파워에이드240) 통째 누락 → 택배0(공짜)·첫 품목 택배 독식**. 신종훈 송장 …0552 실사례. → `_resolve_lines`로 '개' 뒤 콤마 분리해 전 품목 추출.
- **시나리오2 — 175~200ml 30개입 합포가능 규격**: 송장출력엔 품목별로 다른 송장번호로 쪼개져 보여도, 운영상 같은 수령자끼리 **최대 3팩까지 한 박스로 물리 합포**(맨 위 송장 1건 발송). 이게 **openmarket-merge(오픈마켓 합포·도서산간)의 존재 이유** — 같은 주소(수령자)를 합포색그룹(노랑↔초록)으로 물류에 표시하는 그 규칙. 수령자 그룹 합포팩수 P → `ceil(P/3)` 박스, 팩수 비례 배분.

## 합포가능 판정 (규칙성 vs 하드코딩 — 결합)
- 사용자 제공 `175-190합포.xlsx` 60종 분석: **전부 175~200ml(175×42·180×7·190×8·185×2·200×1) + 30개입**. 깨끗한 단일 규칙.
- **그러나 매칭은 관리코드가 정답**: 합포리스트 상품명(`조지아 오리지널, 175ml, 30개`)과 송장 상품명(`[27-64]조지아커피캔오리지날 175 (126)`, ml 생략·박스내품만)이 형식 불일치 → ml 규칙 파싱 불안정. 추출 관리코드(27-64)를 리스트와 대조 = 결정적.
- **결합 채택(사용자 확정)**: 리스트를 `reference/hapo_175_190.csv`(관리코드 60종)로 **박제** → 송장 관리코드 매칭이 실제 판정. 규칙성(175~200ml+30개입)은 신상품 자동후보 추천용(유지보수).
- **데이터 교차검증(송장0616)**: 리스트 O → 조지아175(27-64)·천연사이다190(29-10-01)·광동헛개차180(20-15)·참맑은홍차175(35-05)·알로에(52-55.) = 합포. 리스트 X → 펭귄꽁치400g(14-12)·유동골뱅이230(10-01)·황도400·티오피275*24·헬씨올리고200 = **진짜 다박스**(부피상). 펭귄꽁치 한 수령자 19송장은 진짜 19박스 확인.

## 합포 한도 단위 (사용자 확정)
- "최대 3개" = **30개입 팩 3개**(= 송장 라인 끝 'N개'). ceil(팩수/3) = 박스수.
- 검산: 신종훈 조지아1팩+천연사이다1팩=2팩→1박스 · 오경희 레쓰비+맥콜+참맑은 3팩→1박스 · 광동헛개차 그룹 15팩→5박스(이미 송장당 3팩 → 교정0).
