# 데일리 대시보드 — 이상치 표 이중검수(listing마진+판정) (2026-06-17)

## 무엇
당일 점검(이상치/전체) 표에 **이중검수 컬럼 2개** 추가: `listing마진`(채널 listing 기준 마진율 범위) + `판정`(⚠️ listing도 미달 / listing 정상(일시적) / listing 없음). page-only(0b).

## 왜 (사용자 지적 — 권장가가 현재가보다 낮은 미스터리)
한 행에 **기저가 다른 두 마진**이 섞여 권장가가 직관과 반대로 나옴:
- **당일 실현(daily_margin)**: 매출(net)=실제기입단가×수량(**배송비 수입 미포함**), 택배=실박스수×2700을 **순수 비용** 차감. → 마진율.
- **listing 권장가(cmm compute_listing)**: 정산액=판매가×0.94 + **배송비×0.967(수입 가산)** − 실택배비2700. → 이 기저론 마진이 높게 나와 기준 초과 → 권장가가 현재가보다 **낮게** 나옴.
일시적 쿠폰/할인이 당일 단가를 낮추거나 실박스가 2~3개면 당일만 미달, listing은 정상. **listing이 가격 진실 기준(사용자 확정)** → 당일 미달이라도 listing 정상이면 가격 손댈 필요 없음(일시적), listing도 미달이면 구조적(가격조정).

## 변경
- `core` 변경 없음. `app/pages/0b_데일리대시보드.py` only:
  - `_reco_lookup(pairs, buffer=0.0)` — listing rows의 마진율/기준마진율 집계, listing미달 = 마진율 < 기준 − buffer(당일과 동일 규칙·buffer 분수). 반환 2-tuple→**4-tuple** `(권장가, 현재가, listing마진표시, listing미달여부)`.
  - 표: `listing마진`·`판정` 컬럼 추가(현재가/권장가 뒤). 판정 매핑 {True:⚠️ listing도 미달, False:listing 정상(일시적), None:listing 없음}. `_reco_lookup(..., buffer=float(buffer))`.
  - caption·column_config(help) 보강.
