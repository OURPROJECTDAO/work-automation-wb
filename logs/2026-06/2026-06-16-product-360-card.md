## 왜 / 스펙 (사용자 정의 — "상품이 주인공, 이 상품에 대한 모든 것")
사용자가 일상 ad-hoc로 던지는 질문 = 카드 섹션:
1. "X 상품 어디 채널에서 어느 가격에 매출 얼마나 팔려?" → 채널별 실판매(매출·판매량·실현마진).
2. "어디 채널 어디 채널 올라갔어?" → 8채널 등재 현황 스트립(🟢/⚪).
3. "A 채널에서 마진 N% 보려면 얼마로 올려야 해?" → 채널별 목표마진 역산(임의 N%).
4. "매입가 올랐어 내려갔어? (그래프) 최근 매입 언제?" → 매입가 추이(실입고)+최근입고+장부변경.
+ 재고·소진(두뇌②)도 한 화면(이 상품 모든 것).

## 변경 (page-only · core 무변경)
- 신규 `11_상품360.py`. 전부 기존 순수함수 오케스트레이션 — core 한 줄도 안 고침.
- 섹션 구성:
  - **A 헤더**: resolve_code(4-tier)로 코드유형·규격·매입가(박스/낱개)·박스재고·박스내품.
  - **B 채널 등재+가격+목표마진**: 8채널 저장 listing(`reference/listing_<key>.csv`, `csv_text_to_recs`)
    → 코드 필터 → `cmm.compute(sub, refs, cfg)`. **같은 관리코드 다중 리스팅(N 상이)은 행으로 펼침**.
    컬럼=채널·상품번호·N·현재판매가·매입가·현재마진율·기준여유·**필요판매가(목표 N%)**·인상폭·제한.
    상단 등재 스트립 "🟢채널 ⚪채널"(질문2). number_input 목표마진% 1개로 전 채널 필요판매가 동시 계산.
  - **C 채널별 실판매**: `channel_compare.channel_breakdown`(가격AB 재사용·최근3개월·온라인 그룹·실측 택배).
  - **D 매입가 추이**: `purchases.read_all` 실입고 단가(박스단위 오입력 보정=margin_erosion 동형)
    월별 평균 라인 + 최근매입가/최근매입일/vs6개월전 metric. `price_history` 수정로그 매입단가 변경 expander.
  - **E 재고·소진**: `stockout.forecast`(depletion 3M·cadence 12M) 한 코드 행 — 구간·소진예측일·리드타임·재고금액.
- **목표마진 역산** = 현재 `정산액`에서 유효 정산비율(rate=1−수수료) 역산 → 채널 단일/상품별(배민 BU) 수수료
  모두 정확. 필요판매가 = ((매입가+2700)/(1−목표) − 배송비×settle)/rate + 즉시할인/포인트, **100원 올림**
  (채널마진모니터 권장가와 동일 관례 — 목표 마진 *이상* 보장).
- `streamlit_app.py` 네비 " "(분석) 그룹에 `st.Page 11_상품360.py title="상품 360" 🪪` 등록.
