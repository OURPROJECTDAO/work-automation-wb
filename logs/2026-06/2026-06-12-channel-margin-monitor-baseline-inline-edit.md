# 2026-06-12 channel-margin-monitor: 기준마진율 새 기준 인라인 편집

## 무엇
기준마진율 설정 UX 개선 — offset 라디오(현재 그대로/−1%여유) 제거, **새 기준을 표에서 직접 입력**(data_editor). 기본값=현재 마진율(0.1%p), 사용자가 셀에서 수정.

## 왜
사용자 피드백: "현재 그대로/−1%여유" 프리셋 대신, 바로 편집 단계로 가되 새 기준 기본값을 현재값으로 채우고 거기서 직접 수정. 더 유연(임의 목표값)·충돌도 한 칸 입력으로 자연 해소.

## 변경 (page만, core 무수정)
- `app/pages/6_채널마진모니터.py`: 하단 기준마진율 섹션 교체.
  - 버튼 1개 "🎯 기준마진율 설정 (선택 N)" → propose_baseline(offset=0) → 관리코드별 1행(비충돌=현재값, 충돌=후보 표시·기본 최저값) 스테이징.
  - **st.data_editor**: 관리코드·기존·현재 마진율(기본값 출처)=읽기전용, **새 기준(%)=NumberColumn 편집(percent, step 0.1, 0~100)**. 기본 round(현재*100,1).
  - 저장: 편집값/100 → round(_,3) → update_baseline_csv → commit → cache clear → rerun. 빈 칸 행은 제외.
  - 충돌은 라디오 대신 '현재 마진율' 칸에 `5.3% / 6.8% ⚠️`로 후보 표시 + 안내, 사용자가 새 기준 칸에 직접 입력.
