# 2026-06-11 channel-margin-monitor — 가격변경 양식 G(단위가격사용여부) 기본 N + 빈행 삭제 확인

## 무엇
가격 일괄변경 양식 실사용 테스트 후 보정 2건.
1. **G열(단위가격 사용여부) 규칙**: 출력 행의 G가 비어있으면 'N' 채움, 값 있으면 보존.
2. **빈행/비체크행 삭제 확인**: 플랫폼 업로드 시 변경 행 외 다른 행(빈행 포함)이 남으면 빈 상품행으로 등록됨 → 출력은 헤더(1–5행) + 체크행만 남아야 함. 기존 build가 이미 비체크·빈행 전부 삭제함을 실데이터로 재확인(중간/끝 빈행 주입 테스트 → 출력 빈 데이터행 0). 삭제를 연속구간 묶음으로 효율화.

## 왜
- G 비어있으면 스토어 일괄수정에서 문제 소지 → N 기본값(우리 상품은 단위가격 미사용).
- 변경 행만 남겨야 플랫폼이 빈 행 등록 안 함(사용자 확인).

## 변경 (core)
- CHANNEL_CONFIG[스마트스토어]에 `unitprice_use_col: 7`(G) 추가.
- `build_bulk_price_xlsx`: 체크 행에 `c_up` 있고 G 비면 'N' 세팅(값 있으면 보존). 삭제는 `_ranges_desc`로 연속 구간 아래부터 delete_rows(희소 선택 시 속도↑, 빈행 포함 전부 제거).
- 검증: 행6(원래 N 보존)·행7(None→N), 출력 max_row=헤더5+체크2, 빈행 0. ast OK, 커밋 200.
