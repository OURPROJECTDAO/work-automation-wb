# 2026-06-26 리테일앤인사이트 상품 전체 가격책정 (내부공유 1차)

## 무엇
리테일앤인사이트(B2B 공급처, 상대 요청)에 올릴 상품 전체 가격표 1차 산출 — 챗 네이티브(앱 미탑재).
스마트스토어 상품관리(2026-06-26 10:05 갱신·743행) 기준으로 우리 공급 유닛(박스) 가격·마진을 내부공유용으로 먼저 뽑음.

## 파이프라인 (확정)
1. 유니버스 = 스마트스토어 listing 코드 → **canonical_code(박스 통일)** 로 dedup → 600 유니크(합포 0·PC낱개128·소분69 원박스 환원).
2. 가용재고 필터: 박스재고>0 포함 + **품절(박스재고=0) 중 회전 도는 것 구제**(최근 입고경과일 ≤ 90일 & 매출 소진>0). 재입고 끊김(>90일/입고이력없음)은 제외.
3. 선물세트(product_attributes 식품음료=선물세트) 제외.
4. 기준마진율 = **캐시노트 → 식봄 → 스마트스토어** 체인. ★조회는 **canon 박스코드 + 원래 listing코드 양쪽**(아래 학습).
5. 가격공식(택배비 있는 업체·B2B):
   - 정산액 = 판매가 × (1−0.02)  (수수료 2%·배송비 수입 0)
   - 이익 = 정산액 − 매입가 − 실택배비 2,700
   - 마진율 = 이익 / 정산액  (캐시노트 baseline 정의=이익/정산액과 동일)
   - 판매가 = ⌈ (매입가+2,700)/(1−max(기준마진,0.03))/(1−0.02) ⌉  (100원 올림·3% 하한 클램프)
6. 단위=박스: 매입가=박스매입단가·규격=박스규격·상품명=product_master(ERP).

## 학습
- ★ **baseline_margin은 박스 관리코드뿐 아니라 원래 listing코드(PC낱개·소분)로도 키가 존재.** canon(박스)만 조회하면 PC낱개/소분 상품의 기준마진이 빈 것처럼 보임(99건 오탐) → 캐시노트→식봄→스마트스토어 체인을 **canon + 원listing코드 양쪽**에서 돌려야 정확. 1건만 진짜 스마트스토어 fallback.
- 품절 회전 판정 = slow-moving의 입고경과일 90일 임계 재사용(최근 실입고=합계액>0&수량>0) + 소진>0. 구제 21건 전부 소진>0(매출0 한 건도 없음)이라 robust.
- 리테일앤인사이트는 baseline_margin/CHANNEL_CONFIG에 없는 비설정 채널 → cmm 권장가 공식 오프라인 재현(식봄 행사기획·신규매입 가격책정과 동류).
