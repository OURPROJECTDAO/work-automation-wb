# 2026-06-10 channel-margin-monitor — 채널 추가 레시피 문서화

## 무엇
향후 채널 다수 추가 대비, workflows/channel-margin-monitor.md에 **채널 추가 레시피**(10단계) + **채널별 다른 것/공통인 것** + **정산식 구조 차이 주의** 섹션 추가.

## 왜
이번 세션에 스마트스토어 1채널 + 저장/갱신까지 완성. 사용자: "앞으로 채널 많이 추가". 채널 추가가 CHANNEL_CONFIG 한 세트로 끝나는 경우 vs 정산식 변형이 필요한 경우(배민 관리코드별 수수료 등)를 미리 명문화해 재작업 방지.

## 핵심 (레시피 요지)
- 공통(불변): 코드 4-tier 해석·reference 4종·페이지 UI·listing 저장.
- 채널별: 수수료율·정산계수·실택배비·baseline 컬럼·마진제한 적용·다운로드 컬럼맵.
- 절차: 다운로드 샘플 → cols 매핑 → 정산상수 → baseline 컬럼 확인 → CHANNEL_CONFIG 추가 → 골든검증 → Reboot.
- ⚠️ 정산식: 현재 compute()는 스마트스토어형(판매가×(1−수수료)+배송비×0.967). 배민(관리코드별 수수료)·식봄/쿠팡/알리(계수 상이) 등은 상수론 부족 → settle_fn 훅 검토. 천년경영 마켓별 H식이 1차 참고.
