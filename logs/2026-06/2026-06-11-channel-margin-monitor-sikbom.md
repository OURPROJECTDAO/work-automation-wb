# 2026-06-11 channel-margin-monitor — 식봄 채널 추가

## 무엇
채널 가격·마진 모니터를 두 번째 채널(식봄)로 확장. 코어 일반화(config 축 2개) + 식봄 CHANNEL_CONFIG + 신규 공유 reference(hapo_multiplier.csv) 커밋.

## 왜
로드맵 — 모니터 다채널 확장. 첫 타깃 식봄(사용자 지정). 식봄 다운로드는 스마트스토어와 구조가 달라 config 일반화 필요(배송비 숫자 없음·즉시할인/포인트/바코드 컬럼 없음·N은 외부 참조).

## 입력 분석 (실데이터)
- `식봄.xlsx` (식봄붙여넣기, 485상품): 헤더 r4·데이터 r5+. A상품no(키)·B상품코드·F상품명·S(19)판매단가(=판매가). 즉시할인/포인트/배송비숫자/바코드 컬럼 없음. M=배송비'명'(문자).
- `상품몇개합포인지.xlsx` (3,364행→3,339 dedup): 상품번호·비고(합포량)·오픈마켓. = 골든 '마진율예외처리시트'(O열 XLOOKUP F). 합포량 N(판매배수), 분수 가능(0.166·0.833 등). 식봄 다운로드와 교집합 12건(전부 (빈)채널) — 상품번호 단일키로 충분.
- `식봄_골든.xlsx` (식봄결과페이지 716행): 정산식 추출 — H 정산가=가격×0.93+배송비×0.967, 배송비 G=3000 상수, I 박스매입단가(4-tier), J 마진율=IF(합포<2,(정산가-매입가×합포-3000)/정산가,(...-3700)/정산가), O 합포=마진율예외[상품번호] 기본1, L 권장가=ROUND((...)/0.93,-2).

## 결정 (사용자 확정)
- **식봄을 골든에 충실하게 만들지 않고 웹앱 스마트스토어 구조를 기준으로** 함(수수료율만 채널별). → 실택배비 **2700 flat**(골든 3000/3700 폐기), 권장가 **올림(ceil)**(골든 ROUND 폐기). 상세 ADR 0013.
- 배송비 income측 = **3000 상수**(식봄 다운로드에 숫자 없음; 폐기 대상인 cost측 3000/3700과 별개).
- N(합포량) = 신규 공유 reference `hapo_multiplier.csv`(상품번호 키). config `n_source="ref"`.
