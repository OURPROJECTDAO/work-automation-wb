# 2026-06-19 두뇌④ ⑦ 매출목표 라벨 + 나들 floor (나)

## 무엇 / 왜
⑦ 상품 매출목표 = 작은 상품(어디서도 안 큼)을 볼륨 푸시로 키우는 라벨. 설계 인터뷰로 단순화·함정 제거.

## 설계 확정 (사용자)
- 원안 "best 채널(나들 포함) 매출 따라가기"는 **나들이 바닥 채널이라 = 모든 채널을 나들까지 무한 인하** → 폐기.
- 살아남은 건 **100만원 절대 floor**: best 관리채널 월매출 < 100만 = "매출목표 미달". 작은 상품에만·단방향·바닥 고정 → 폭주 없음.
- (나) 선택 = 표시 + 자동 인하 제안(측정 데이터는 어차피 쌓임).
- (나)의 "얼마나 내릴지" 바닥 = **나들 마진율**(ADR 0026 §1 하한) → ⑦과 나들 floor 한 덩어리.

## 규칙
- best 관리채널 월매출 ≥ 100만 → **목표 라벨 없음**(베이스 4분면+측정 루프가 최적).
- 미달(<100만) 상품:
  - 고마진/베이스근처/proven 셀 → **나들 마진까지 절반스텝 인하**(거리의 절반·측정 후 또 절반/되돌림). 역마진 방지(나들 anchor라 기본 보장).
  - 나들이 그 상품 안 팔아 마진 없으면 → fallback **−1%p 테스트 인하**.
  - 이미 저마진(<베이스) 셀 → **🔴 사람판단**(가격 외 요인 의심: 노출·핏). 자동 인하 안 함.
- 전부 측정 루프 제동(악화→되돌림). 나들 = floor 계산엔 포함(제외 전 1패스), 작업목록엔 제외.

## 변경 (work-automation-app · core+page)
- `core/intelligence/margin_optimizer.py`: 상수 `TARGET_FLOOR=1,000,000`·`TEST_CAP=0.01`. `recommend_code(cells_code, nadl_margin=None)` — small=best 월매출<floor, 미달 override(나들/저마진/fallback) + `목표` 컬럼. `worklist(cells, codes, nadl_map)` — 관리코드별 나들 마진 주입(`nadl_map.get(_nfc(code))`).
- `app/pages/13_…py`: `build_prod` → **나들 제외 전** 관리코드별 나들 순마진(_순/판매금액) 1패스 계산 → `(prod, nadl_map)` 반환. `build_worklist`가 nadl_map 전달. tab2 build_prod 언팩. 작업목록 표에 `목표`(📉미달) 컬럼·KPI 캡션에 미달 건수.

## 검증 (합성)
- small·나들4%: 캐시 9%→6.5%(나들까지 절반)·저마진(식봄6.4·알리3)→🔴·전행 📉미달 ✓
- small·나들없음: −1%p 테스트(8%→7%) ✓
- big(best 300만): 목표 미적용·기존 베이스 로직(알리 고마진→베이스로) ✓
- ast 3파일 OK.
