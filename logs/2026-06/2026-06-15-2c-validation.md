# 2026-06-15 intelligence-layer — 2C 가정 실증 (master 매입가 ↔ 매입현황 = 독립·수동)

## 무엇 / 왜
두뇌① 강화(2C 조기경보)의 핵심 가정 검증. 사용자(입력자 아님) 질문: "매입현황 입력 시 상품관리(master) 매입가가 자동으로 변하나, 따로인가?" → 데이터로 직접 확인(KB는 진실원천 아님). 매입단가 수정로그(price_changes)와 실입고 단가(buyin)를 같은 관리코드로 대조.

## 결론: 자동 연동 아님 — 독립·수동(반자동, 수동 트리거)
매입은 매입대로 들어가고, 운영자가 입고 단가를 **보고 판단해 master 매입가를 손으로 수정**한다. 입고가 master를 자동으로 바꾸지 않음.

## 근거 (실측, 겹치는 기간 2025-06-16~2026-05 / master 매입단가 수정 1,539건 대조)
- **수정후 값 = 직전 ±7일 실입고 단가 일치 = 35%뿐.** 자동이면 ~100%여야. 입고가 그대로 안 들어가고 협의가·마진 감안한 다른 값을 넣음.
- **시차 큼**: 수정일 당일 입고 41% · ≤7일 77% · **>30일 8%** · 중앙값 1일·평균 9.9일(긴 꼬리). 자동이면 전부 당일.
- **사람 손 흔적**: 23-19(웅진초록매실) master `4000→5000→476→4762` 오타·정정 / 22-04(비락식혜) 같은 날 다회 수정·익일 `2462→1250→2529` 엎치락. 자동 시스템은 이런 값 안 만듦.
- 31-24(닥터페퍼): 7월 입고 활발(633/565)인데 master는 6/9/4월에만 띄엄띄엄 → 입고마다 반영 안 됨.

## 2C 조기경보에 주는 함의 (가정 확증됨)
- "실입고 단가↑ ∩ master 미수정" **시차 윈도우 실재**(>7일 미반영 23%·>30일 8%). 그동안 장부마진 과대 → 운영자가 master 수정 순간 마진 급락.
- master가 실입고가를 정확 추종 안 함(일치 35%) → 옛 낮은 단가에 정체 가능 = 과대 장부마진의 또 다른 원천.
- → **2C = 운영자보다 먼저 "곧 마진 떨어질 상품" 포착.** 신호 실재 확인.

## ★ 구현 키 함정 (다음 구현 때 필수)
- **price_changes 관리코드 = 트레일링 하이픈**(예 '31-24-'·'22-04-'). buyin/product_master 관리코드(풀코드)와 매칭 시 **트레일링 `-` 제거 정규화 필수**(`re.sub(r"-+$","", NFC(c).strip())`). 안 하면 조용히 미매칭. 상품코드(6자리)는 buyin에 없어 관리코드 키 사용.
- 단가 단위: buyin 단가·price_changes 매입단가 둘 다 **낱개 단가**(비교 OK). price_changes 앵커=master 낱개 매입단가[8](95.4%, 기존 1a 확정).
