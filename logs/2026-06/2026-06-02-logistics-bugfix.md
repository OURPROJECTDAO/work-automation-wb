# 로그: 발주서출력업무 버그픽스 + 골든 검증

## 무엇
실 데이터 첫 실행에서 나온 버그 2건 수정 + 골든 대조 검증.

## 버그 1: Phase2 int64 캐스팅 오류
- 증상: `Invalid value '1.25' for dtype 'int64'` (reconcile_stock).
- 원인: _필요수량 컬럼이 총수량(정수)에서 int64로 초기화 → 낱개 배수 곱(10×0.125=1.25) 대입 시 실패.
- 수정: `_필요수량` 을 .astype(float) 명시.

## 버그 2: 발주자료 아카이브 열 위치 불일치
- 증상: 실제 아카이브(8열, 중복제거 전, 155행) vs 웹앱(10열, 중복제거 후, 105행).
- 원인: 완성된 phase1_df(구분/규격 추가+중복제거)로 아카이브 생성했음.
- 정정: 발주자료 = 정제+셀나누기까지만 된 원본 8열 스냅샷(개별주문 보존, 중복제거 전).
- 수정: run_phase1이 archive_df(8열 스냅샷) 별도 반환 → generate_archive_xlsx가 원본 헤더(어드민 옵션 띄어쓰기)로 출력.

## 검증 (골든 대조)
- 골든 내장 상품관리(4222행)로 재실행 → 품절목록 10/10 정확 일치.
- 물류팀 106행 = 골든 106행, 병합셀 22개, C열 숨김 일치.
- 업로드 상품관리(Exp260602_081321)는 재고 적은 다른 시점이라 품절 13행 (데이터 차이, 정상).
