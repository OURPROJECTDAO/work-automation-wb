# 2026-06-19 두뇌④ 측정 루프 하드닝 — 커버리지 게이트 + 일수 정규화

## 무엇 / 왜
측정 루프(같은 세션 구현)를 실데이터 적재 cadence와 정합. 사용자 확인: 매출자료는 **익월 초** 적재(예 7월초 6월분) → 월별 갭. 갭 자체는 비동기라 무해하나, v0 `ready=벽시계 30일 ∧ post_개월>0` + `run-rate=÷달력월개수`는 **월말 결정 + 부분월 적재** 시 큰 편향(예 7/25 결정·8/4 측정 시 7/26~7/31 6일치만 적재됐는데 `/1개월` → 월순이익 ~80% 과소 → 거짓 "악화"·멀쩡한 결정 되돌림 위험).

## 변경 (work-automation-app · core+page only)
- `core/intelligence/margin_optimizer.py` measure():
  - **커버리지 게이트**: ready = (적재 최신거래일 data_max − ts) ≥ MEASURE_MIN_DAYS. 벽시계(today−ts) 아님 → 월별 적재 갭에 안 휘둘림(자료 도착 전엔 측정대기, 도착하면 자동 측정 가능).
  - **일수 정규화**: 월순/월볼 = ts 이후 거래 합 ÷ (측정일수/30.4). 달력월 개수(post_개월) 아님 → 부분월·월 어디에 떨어지든 사과 대 사과.
  - ready인데 post 매출 0 = 월순 0(판매 붕괴 신호·악화), 미ready = 측정대기. 신규 컬럼 `측정일수`(=cov_days). data_max는 prod 프론티어(전 셀 공통).
- `app/pages/13_…py` 측정 탭: ready표·대기 expander에 `측정일수` 노출 + 게이트/일수정규화 설명. 대기 안내("다음 월 적재되면 자동 측정 가능").

## 검증 (합성 4케이스)
- 월말결정+8월완결(cov 31) → ready·일수정규화 월순 608K → 개선 ✓
- 월말결정+7월만 부분월(cov<30) → **측정대기**(거짓악화 방지) ✓ ★핵심
- 오늘결정·data 5월말(cov 음수) → 측정대기 ✓
- ready+매출0(cov 60) → 월순0 → 악화 ✓
- ast 3파일 OK.

## 효과
- 월초 결정 거의 무해했던 부분, 월말 결정 심각 편향 → **둘 다 해소**. "30일" 의도는 유지(적재 30일분 기준)·자료 도착에 맞춰 늦게 측정.
