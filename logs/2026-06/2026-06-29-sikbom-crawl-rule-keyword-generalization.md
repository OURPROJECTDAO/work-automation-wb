# 2026-06-29 식봄 경쟁가 크롤링 규칙 — 검색어 자동 일반화 + 멀티키워드 병합

## 무엇
식봄(foodspring) 경쟁가 조회 스크립트(`tools/sikbom_price_lookup.py`)에 **검색어 일반화 규칙**과 **멀티키워드 병합**을 추가. 우리 통짜 등재명으로 검색 시 우리만 걸려 '경쟁없음'을 오판하던 구멍을 막음.

## 왜
사용자 관찰: "진양 죽순 4호 400g 1개" 통짜 등재명으로 조사하면 우리만 떠서 "우리 최저/경쟁없음"이 나오는데, "진양 죽순 400g"으로 치면 경쟁사가 꽤 뜬다 → 크롤링이 경쟁사를 통째로 못 보는 구조적 구멍.
- 라이브 재현(2026-06-29):
  - `"진양 죽순 4호 400g 1개"` → **1건(태동유통 우리만)** → '경쟁없음' 오판
  - `"진양 죽순 400g"` → **26건** (더상상 2,700·온국민 2,800·현대그린푸드·CJ프레시웨이·우주식품·온퀵·리바이·싼대로마트·시장바구니·더착한푸드몰 + 박스 X4/X24)
- 원인: 식봄 검색은 ACCURACY 매칭. "4호"(판매자 전용 캔 호수 표기)·"1개"(낱개 꼬리)가 붙어 검색이 분절 → 우리 통짜 제목엔 우리만 매칭.
- 이번 죽순 건은 우리(2,500)가 단캔 기준 진짜 최저였지만, 다른 품목에서 경쟁이 더 쌌으면 놓쳤을 위험. 7차 학습("맛 키워드 0건 ≠ 경쟁 부재")의 일반화 버전 = 검색어 자체가 너무 좁으면 경쟁 부재로 오판.

## 변경 (tools/sikbom_price_lookup.py)
1. **`normalize_keyword(raw)`** 신규 — 통짜 등재명 → 일반화 검색어:
   - 괄호 처리: 속성만(상온/실온/냉장/냉동/택배/직배송/무료배송/대용량/업소용/벌크/낱개/개당/ea)이면 제거, 브랜드·규격이 들었으면 **언랩**(괄호만 제거·내용 보존). 예 "죽순캔(진양 400g) (상온) 1개" → "죽순캔 진양 400g".
   - 호수 제거: `\d+호` (4호 등 판매자 전용 표기·검색 분절).
   - 낱개꼬리 제거: 단독 "1개"/"낱개" — 단 멀티팩(X24·24개·*24개)은 보존(`(?<![xX*\d])` 룩비하인드).
   - **코어(브랜드+품목+규격 g/ml/kg/L) 보존** — 브랜드만 남기는 과일반화는 6차 오매칭(사조해표→양조식초)이라 안 함.
2. **`crawl_merge(keywords, jar)`** 신규 — 여러 검색어 각각 크롤 → `(vendor,name,sale)` 키로 dedup 병합(nid 동치). 누락 방지.
3. **`main`** — 검색어 집합 = 준 검색어 + (자동)일반화 + (수동)`--extra`. `--exact`로 자동 일반화 끔. 헤더에 사용 키워드·건수 표기.
4. **`spec_hint`** — 멀티팩(X24·*24·24개입·24입)을 `x24`로 채집해 규격에 표기. 단캔 `[400g]` vs 박스 `[400g x24]` 구분 → AI가 동일규격만 비교.
5. docstring 사용법 갱신(일반화 규칙·--extra/--exact·salePrice 비교기준 명시).

## 검증
- `ast.parse` OK.
- normalize 단위테스트 전건 정확:
  - "진양 죽순 4호 400g 1개" → "진양 죽순 400g"
  - "진양 죽순 400g*24개/box" → 그대로(멀티팩 보존)
  - "코카콜라 제로 355ml 24개입" → 그대로
  - "죽순캔(진양 400g) (상온) 1개" → "죽순캔 진양 400g"(브랜드+규격 보존·속성 제거)
  - "동원 참치 라이트스탠다드 100g (택배배송)" → 속성 제거
- 라이브 end-to-end: 통짜 등재명만 입력 → 자동으로 "진양 죽순 400g" 병합 검색 → **25건**(dedup), 단캔/박스 규격 구분, 우리 2,500=15.8%·경쟁최저 2,700(더상상)=18.7% 정상.
- `--exact`(1건만)·`--extra "죽순캔 400g, 진양죽순"`(4키워드 47건 병합) 옵션 정상.
