# 2026-06-09 easyadmin-register 워크플로우 정립 + 글로벌하베스트 15개 작성

## 무엇
신규 채널 **이지어드민**(정산 관련, 판매 플랫폼 아님 — 모든 신규상품 필수 경유) 상품등록 규칙을 예시참고로 역설계 + 실제 배치(글로벌하베스트 15개) 업로드 파일 생성. workflows/easyadmin-register.md 신설.

## 왜
스마트스토어 등록한 15품목을 이지어드민에도 등록해야 함. 낱개/박스 입력 형식이 달라 규칙화 필요.

## 변경 / 확정
- 기입 대상 = `상품등록(실제 업로드파일)` 2행 `기입` 표시 7열만(A·B·C·D·E·AB·AC). 69열·3시트·서식 보존, 데이터 2행부터.
- 예시참고는 등록 후 export 포맷(열 배치 다름) → 헤더명 매핑으로 규칙 도출.
- A 상품명: 낱개 `[PC코드][낱개1개][이름ㅁ1개입ㅁ]`(레거시, ㅁ=U+3141, 이름=정제명-' 1개'), 박스 `[관리코드]{정제명}`.
- C=A에서 맨앞 [코드] 제거 · E=C · D(낱개)=A · D(박스)=`[관리코드]-{정제풀네임} [용량*박스내품]`.
- B=20010 고정 · AB=4 고정 · AC=낱개 박스내품 / 박스 합포장(기본1).
- 데이터: product_master(박스내품·규격), 정제명=넣어야할것(스마트스토어 등록명).

## 검증
- product_master에서 15코드 전부 조회(낱개 PC→상품코드 매핑, 박스 관리코드).
- 생성 .xlsx 재로드: 69열 유지·15행(2~16)·헤더 일치·기입열만 채워지고 비기입열 빈칸(F2=None) 확인.
- 사용자 확인: 낱개·합포(AC) OK, B=20010 OK, 박스 D는 풀네임 전체로 수정 반영.
