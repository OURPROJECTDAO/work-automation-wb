# 2026-06-09 smartstore-register 워크플로우 정립 + 글로벌하베스트 배치 생성

## 무엇
신규 워크플로우 smartstore-register(스마트스토어 상품 등록)을 처음부터 한 바퀴 돌려 규칙 확정 + 실제 배치(글로벌하베스트 15개) 일괄등록 .xlsx 생성. 운영 방식 ADR 0009 채택.

## 왜
상품 등록 업무 자동화 시작. 약 7~8채널 확대 예정. 결정적/판단 혼합 워크플로우라 운영 형태부터 결정.

## 변경 / 확정
- 두 모드(낱개 PC+상품코드 / 박스 관리코드) + 매입가·규격suffix·합포N 규칙.
- 판매가 공식 실물검증: 판매가=⌊(매입가+2500)/[(1−0.06)(1−마진)]−3000⌋(100원내림). 수수료는 (판매가+배송비)에, 마진 분모=정산액, 디폴트 마진 10%. (3400/3300 원단위 일치)
- 상품명 정제 규칙(띄어쓰기·슬래시/내부수량 제거·브랜드앞·모드 suffix), AI 제안.
- 합포장→배송비템플릿 N(박스=합포장, 낱개=합포장×박스내품) + 스마트스토어 N→코드 17종 표.
- 카테고리: reference/smartstore_category_food.csv(630, 식품) 룩업 + 애매 시 웹검색. 15개 매핑 완료.
- 이미지 URL 패턴(A1/B1), 확장자 실검사 필요(이번 jpg).
- 양식 13컬럼 + 셀 타입/서식 스펙(카테고리=텍스트, 배송비=숫자, 원산지=@텍스트 03). 생성=양식복사+서식보존.
- 부가세는 고정 아님(과세/면세 사용자 지정), 배송비코드도 사용자제공(채널별).

## 검증
- product_master에서 관리코드↔상품코드 매핑·매입가 실조회로 공식 검증.
- 생성 .xlsx 재로드: 헤더2행 일치, 93열 유지, 13컬럼 전부 타입/서식 == 양식(AD=@, B=str, AI=int) OK, 15행(3~17).
- 카테고리 애매 품목(우롱티/그린티/요구르트스파클링/누룽지과자) 웹검색으로 성격 확인 후 매핑.
