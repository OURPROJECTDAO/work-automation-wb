## robust 키 규칙 (데이터로 확정·검증)
- **이름** = 끝 1자 제외. 실측 마스킹 끝글자수 분포 = 1자 137,871 / 2자 11(0.008%) → 끝1자 가정 확정. 괄호 주석 제거·공백 제거·끝 `*` 정리.
- **전화** = 뒤 4자리(모든 마스킹 패턴에서 끝4 생존). **안심번호(`9999-9999-9999` 4-4-4 가상) 제외** — 채널셋(쿠팡·11번가·멸치쇼핑·셀러허브·옥션·G마켓·위메프) 전체 + per-row 4-4-4 감지.
- **주소(고객키)** = 숫자/`*` 토큰 전까지 앞 토큰(시군구+도로명, 최대 3토큰). 마스킹은 꼬리만 → 앞부분 생존.
- **합포박스키** = 채널+풀이름(NFC)+풀주소(NFC)+발주일. 박스=1회성·다운로드 내 일관 → 풀값 정밀.
- 해시 = `sha1(salt|C or B|채널|조각…)[:16]`. **솔트=st.secrets[data].customer_key_salt** (이번 세션 생성·채팅 전달·repo/KB/로그 미기재). 채널명 prefix(채널 내 키). 원본 PII 키 생성 직후 폐기.

## 충돌 검증 (over-merge)
- 전 채널 키당 신원 중앙값=1·p95≤3. **전화 포함 채널 깔끔**(식봄/배민/올웨이즈/제이티/캐시노트 max 4~7) vs **안심 채널 꼬리 큼**(G마켓52·쿠팡44·스마트31·멸치26 — 이름+주소만). 비-안심 over-merge는 대부분 마스킹 변이 정상 병합(전화뒤4+주소+이름앞 일치=동일인). 안심 채널 큰 꼬리=진짜 충돌 → **고카디널리티 키 저신뢰 플래그**(향후 두뇌③ 가중↓). 사용자 동의 트레이드오프(식별력 일부 포기↔시간 일관성).
