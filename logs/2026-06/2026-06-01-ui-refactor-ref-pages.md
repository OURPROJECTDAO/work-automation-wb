# 로그: 기준데이터관리 UI 리팩토링

## 무엇
기준데이터관리 페이지의 분류/단가 데이터를 사이드바 하위 폴더(nested pages)로 분리.

## 왜
SKU단가표를 도서산간 탭들과 같은 줄에 두니 어떤 워크플로우 데이터인지 불명확.
사이드바에서 워크플로우명으로 직접 보이게 해달라는 요청.

## 변경 (순서대로)
1. `2_기준데이터관리.py` 단일 파일에 섹션 분리 시도 (st.subheader + divider) → 즉시 교체됨.
2. Streamlit nested pages 구조로 전환:
   - `app/pages/2_기준데이터관리.py` 삭제
   - `app/pages/2_기준데이터관리/1_분류기준데이터.py` 생성
   - `app/pages/2_기준데이터관리/2_단가기준데이터.py` 생성
   - `requirements.txt`: streamlit>=1.35 → >=1.36 (nested pages 지원 버전)
3. 서브페이지 파일명을 workflow명으로 변경:
   - `1_분류기준데이터.py` → `1_오픈마켓합포도서산간확인.py`
   - `2_단가기준데이터.py` → `2_온누리양식_발주서.py`

## 최종 사이드바 구조
```
파일처리
기준데이터관리
  └ 오픈마켓합포도서산간확인
  └ 온누리양식_발주서
대시보드
```

## 검증
- GitHub API로 `app/pages/2_기준데이터관리/` 디렉토리 내 파일 2개 확인.
- 공통 헬퍼(render_tabs, _gh_put_csv 등)는 각 파일에 인라인으로 포함.
