# 2026-06-16 채널 비교(가격 A/B) v1 — 관리코드 × 채널 실현마진/볼륨/가격 (두뇌③ 서술)

## 무엇
두뇌③ 채널 가격 A/B의 **v1(서술)** 구현. 관리코드 1개를 골라 **채널별 실현마진·판매량·매출·가격**을 가로로 비교하는 신규 페이지 `가격 A/B`(10_가격AB.py) + 코어 `core/intelligence/channel_compare.py`.

## 왜
사용자 확정: A/B의 분석 단위는 "채널"이 아니라 **(상품 × 채널) 셀** — 상품 기준으로 채널을 분류("어디서 많이 파나 / 어디서 비싸게 파나"). grain=**관리코드 단건**(마진·매입가 정확), v1=**현재 비교판(서술)부터**, 탄력성 예측은 후속(데이터가 준실험·행사로깅 미적립이라 신뢰구간 넓음).

선행 정리(사용자와): ① 레버는 마진율이 아니라 **판매가**(채널마다 수수료 달라 같은 마진율≠같은 가격) ② 채널 비교 마진율은 **분모=매입가**가 정직(매입가는 채널 무관 고정 → 차이가 분자에만) ③ 마진율만으론 "비싸게 팔 수 있는 지점" 안 보임 → **낱개당 절대이익** 병기.

## 변경
- **core/intelligence/channel_compare.py (신규)**:
  - `compute_online_margin(view, ship_rate, unit, use_actual)` — 대시보드 _render_online_margin 과 **동일 계산**을 순수함수로 추출(추정 k=채널 00-12÷추정송장, 실측=ship_alloc.attach_actual_ship 우선·NaN이면 추정 fallback). _택배/_매입(=판매금액−판매이익)/_순/_실측 부여. ★k는 view 전 관리코드로 산출돼야 정확 → **코드 필터 전에 호출**.
  - `channel_breakdown(prod, code, ea_price)` — 한 관리코드의 채널(상호명)별 표: 채널·마진율(순이익÷매입가)·낱개이익(순이익÷판매량)·매출·판매량·정산단가(매출÷판매량·net낱개)·노출가(EA)·택배(실측/추정/혼합). 매출 desc.
  - `build_ea_price_agg(orders)` / `ea_price_lookup(agg, code, months)` — EA 판매가[30]를 (상호명,관리코드,ym) 가중집계 → 기간 가중평균 노출가(gross). 판매처→상호명=ship_alloc.ea_to_sangho.
  - `CHANNEL_LABEL`/`label()` — 온라인 상호명→친근 채널명(스마트스토어·ESM(G마켓·옥션)·식봄·캐시노트·쿠팡·배민상회·올웨이즈·알리·제이티유통), 외=상호명 그대로.
  - `months_in_range(d_start, d_end)`.
- **app/pages/10_가격AB.py (신규)**: 기간(기본 최근 3개월)·택배단가 토글·실측 토글 → 온라인 거래처('온라인' 그룹) view → 관리코드 검색/selectbox(매출 desc·채널수 표시) → KPI(총매출·총판매량·채널수) + 비교 표 + 막대 2개(매출·마진율) + CSV. cached 로더(load_sales_min·load_orders·load_ship_rate·load_ea_agg·load_group_map). page-only이나 **core 신규 import → 첫 배포 후 Reboot app 1회**.
