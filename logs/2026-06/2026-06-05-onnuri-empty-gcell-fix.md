# 2026-06-05 온누리 발주서 — 빈 G열(합계) 셀 값 누락 버그픽스

## 증상
sharedString 픽스(2026-06-04, zipfile 직접조작) 이후 출력 파일의 G열(합계:판매가) 값이 안 잡힘(빈칸). 일부 행 데이터도 어긋남.

## 근본 원인
`_patch_column_values`의 정규식 `<c r="G2"[^>]*>.*?</c>` 가 **`</c>` 닫는 태그를 가진 셀만** 매칭.
신선한 템플릿의 합계 셀은 비어 있어 두 형태로 존재 → 둘 다 실패:
1. **자체닫힘 빈 셀** `<c r="G2" s="3"/>`: `</c>`가 없어 `.*?</c>`가 **다음 행 셀의 `</c>`까지 과매칭** → 그 사이 셀(다음 행 데이터)을 삼키고 값도 엉뚱한 위치에 → 데이터 손상.
2. **셀 자체 없음** (행에 `<c r="G..">` 미존재): `re.search`가 None → 값 조용히 건너뜀 → G열 빈칸.
- 기존 값이 있던 셀만 정상 동작. 2026-06-04 검증이 "99001→99001" 즉 **이미 값 있는 파일**로 이뤄져 못 잡힌 회귀.

## 수정 (work-automation-app / core/workflows/onnuri_order.py)
`_patch_column_values` 재작성 + 헬퍼 2개 추가:
- 매칭 정규식을 `<c r="G2"([^>]*?)(?:/>|>(?:(?!</c>).)*?</c>)` 로 변경
  → 값셀·자체닫힘 셀 모두 매칭, 템퍼드(`(?!</c>)`)로 다음 셀 침범 방지.
- 매칭 시: 자체닫힘이든 값셀이든 `<c r=".." s=".."><v>값</v></c>` 로 치환(스타일 보존).
- 미매칭(셀 없음) 시: `_insert_cell_into_row()` 가 해당 `<row>` 안에 **열 순서를 지켜** 새 셀 삽입.
- `_col_letter_to_num()` 헬퍼 추가(A→1, G→7).
