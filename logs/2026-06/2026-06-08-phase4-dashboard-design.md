## 성능 실측 (3개년≈50만행 모사)
- `pd.read_excel`(openpyxl) 50만행 = **65초/피크 548MB**(위험). calamine 7.7초/649MB. read_only 스트리밍 24초/+222MB.
- parquet 변환 후: 50만행 9MB·재로딩 0.35초·category dtype 상주 58MB(151→58, 61%↓). 집계(groupby/pivot) 전부 <100ms.
- 1년치(17만행): calamine 7.7초, parquet 3MB/23MB.
- Streamlit Community Cloud = **RAM 1GB/앱(동시접속 공유)** → 큰 xlsx 직접 read_excel 금지.

## 확정 설계 (decisions/0006)
- 수집: 연도별 부트스트랩(1년씩) + 일/주/월 증분 → **앱이 calamine 파싱** → **월별 parquet 파티션** 누적.
- 중복: 고유 전표ID 없음 → **날짜구간 통째 교체**(파일 min~max 날짜를 master에서 지우고 재삽입, 최근 다운로드 우선).
- 로드: `@st.cache_data` 1회 로드.
- 저장위치: master=PII(거래처 매출) → public app repo 금지. (A)Drive 추천 / (B)세션 보유 — **사용자 추후 택**(잠정 Drive).
- 거래처 그룹: **앱 내 기준표(B안)** — 온/오프라인 등 자유 배정, 오픈마켓/스마트스토어/쿠팡 자동추천(기준데이터관리 패턴).
- 물류량 = **수량 ÷ 박스내품** (reference/product_master.csv 또는 unit_list.csv, 관리코드 조인).
- 식품/음료 구분 = product_master.csv 대분류/중분류(예: 음료-B동)에서 끌어올 가능성(잠정, 별도표 불필요할 수도).
- 화면: 파워BI 스타일 — KPI(매출·누적·이익률·물류량) + 멀티연도 월별 매출 비교 + 이익+물류량 콤보(이중축) + 상품분류 상세표 + 슬라이서(연도/그룹/식품음료/월범위/상호명) + 별도 [거래처 그룹 관리] 탭.
