# 2026-06-10 ESM(G마켓) 업로드 사후처리 — 합포장·관리코드 백필

## 무엇
양식 없이 먼저 올렸던 글로벌하베스트 G마켓 등록건의 사후 기록 파일(`업로드_사후처리.xlsx`, 지마켓 export 15품목)에
D열=합포장 개수, E열=관리코드 백필. 산출 `업로드_사후처리_완성.xlsx`.

## 왜
입력 양식(v2) 도입 전에 처리해서 합포장·관리코드 기록이 누락 → 소스 시트로 역채움.

## 변경 / 방법
- 소스 = `[소스]글로벌하베스트.xlsx`(사용자 업로드 = Drive 상품등록/esm/ 동일본). 컬럼 A상품코드·B관리코드·C상품명(원본)·D업로드구분(전부 박스)·E합포장·F규격.
- **조인 키 주의**: G마켓 export C열은 *정제* 상품명, 소스 C는 *원본* 압축명 → 직접 문자열 조인 불가.
  규격(용량 g/ml + 박스내품 개수 `*N`) + 맛/종류 키워드 char-overlap 스코어로 자동 매칭, 15건 전수 검수(개수·맛 일치). 누룽지 무가당/달콤, 새우칩 갈릭/게맛살/청양마요 동일규격 구분 OK.
- D=소스 합포장(1, 라인프렌즈만 2). E=소스 관리코드(@텍스트, 앞자리0 보존). 전부 박스라 상품코드 아닌 관리코드.

## 검증
- 매칭 15건 모두 score 만점 + post '개수' == 소스 규격 count 일치. 재로드로 D/E 값·E열 @서식 확인.
