# 2026-06-09 기준데이터 편집표 전수 — 검색+동적높이+merge-back

## 무엇
기준데이터관리 4개 페이지의 모든 인라인 편집표에 **검색 필터 + 동적 높이 + 키 무관 merge-back** 적용.
천년경영(배민수수료 780행·소분목록 356행)은 `head(50)`+CSV통째교체-only → **인라인 편집 가능**으로 전환(전체 교체는 expander로 유지).

## 왜
사용자 보고: 발주서출력 소분코드 표가 "상위 50개만 보이고 편집 불가".
원인 — ① st.data_editor는 height 윈도우만 렌더+스크롤, ② 셀 편집 시 스크롤이 top으로 리셋(streamlit #10181) → 수백 행에서 아래쪽 편집 사실상 불가, ③ 천년경영은 아예 head(50) 미리보기+업로드교체만이라 50행 이후 손도 못 댐.

## 변경
- `app/pages/2_기준데이터관리/{1_오픈마켓합포도서산간확인,2_온누리양식_발주서,3_발주서출력업무,4_천년경영업로드}.py`
- 공통 헬퍼 `_edit_with_search(df, name, key_col)` 각 파일에 삽입:
  검색창(전열 contains, regex=False, fillna('').astype(str)) → 필터된 행만 data_editor(num_rows="dynamic", 높이 min(38*(행+1)+3, 560)) → 저장 시 `pd.concat([df[~mask], edited])`로 merge-back → key_col 빈행(없으면 전부빈행) 정리.
- 대용량(도서산간 1만+·규격 4366)은 검색 **미리보기(읽기전용)** + 파일 교체 유지(인라인 편집 안 함).
- 파일1·2는 동일 render_tabs 공유 구조 유지. 파일4는 _section을 편집표+업로드expander로 재작성(컬럼검증 유지).
