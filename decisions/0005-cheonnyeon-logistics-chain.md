# 0005 — 천년경영업로드: logistics 체인 + 분류표 공유

날짜: 2026-06-04
상태: 채택

## 맥락
천년경영업로드 워크플로우는 (1) 발주자료, (2) 멸치쇼핑 분류표(구분 판정)를 필요로 한다.
발주서출력업무(logistics-order)는 이미 (1) 발주자료 아카이브를 산출하고, (2) `logistics_classification.csv`(멸치쇼핑 분류표)를 기준데이터로 관리한다.

## 결정
- 천년경영의 발주자료 입력 = logistics 발주자료 아카이브 앞 7열. 두 워크플로우를 **체인**으로 본다(logistics 출력 → cheonnyeon 입력).
- 멸치쇼핑 분류표는 **단일 출처**로 `logistics_classification.csv` 공유. 천년경영은 읽기 전용, 관리(추가/수정)는 발주서출력업무 기준데이터 페이지에서만.
- 천년경영 전용 신규 기준데이터(배민상회 수수료율·소분목록)는 별도 CSV로 분리해 천년경영 기준데이터 페이지에서 관리.

## 근거
- 분류표 이중 관리 시 두 파일이 갈라져 구분 판정이 어긋날 위험 제거. 골든 worksheet 215코드가 logistics 분류표와 100% 일치함을 확인 → 공유 안전.
- 사용자 작업 흐름과 일치(이미 발주서 출력에서 발주자료를 받아 업로드에 쓰던 수작업을 그대로 자동화).

## 영향
- 분류표 스키마/위치 변경 시 두 워크플로우 동시 영향 → workflows 양쪽 문서에 공유 명시.
- 천년경영은 logistics 아카이브 포맷(앞 7열 순서)에 의존. logistics 아카이브 컬럼 순서 변경 시 천년경영 parse_baeju 확인 필요.
