# 0015 — ESM(G마켓) 마진 모니터: 17.5% 단일 + 지마켓 필터/A중복제거

날짜: 2026-06-12 · 상태: 채택

## 맥락
채널 가격·마진 모니터에 ESM(G마켓) 추가. ESM=G마켓+옥션 통합 채널로, 상품관리 다운로드가 1회 500상품 한도라 여러 배치(11/22/33)로 받은 뒤 사용자는 수기로 합치고 F열 "지마켓"만 추려 골든을 산출해 왔다. 골든(esm결과페이지)은 외부참조([1]esm전체다운로드) 수식 전용이며 N 참조가 `#REF!`로 깨져 있다.

## 결정
1. **수수료 = 17.5% 단일.** 골든 총정산가 = `가격×0.825 + 배송비×0.967` → (1−수수료)=0.825 → 17.5%가 실제 정산을 정확히 재현. 다운로드 O열 판매이용료(13/11/9%)는 카테고리 수수료라 정산 수수료 아님 → 미사용. 골든 권장가 분모 0.83·헤더 "17% 기준"(=17%)은 시트 내부 불일치(0.825 vs 0.83)이며, **정산(=마진율 신호의 핵심)을 우선**해 17.5% 채택(사용자 확정). 권장가는 표준 2700·ceil이라 어차피 골든과 의도적 차이.
2. **키 = A(마스터상품번호).** 골든이 esm상품번호·관리코드·가격을 전부 A로 XLOOKUP. hapo N도 A로 매칭(esm∩A 178, ∩B 0). baseline·매입가·floor는 E(판매자관리코드).
3. **지마켓만 모니터 + A 중복제거 (parse 자동).** 새 config knob `include_row_if_col_value`{col6,value"지마켓"} + `dedup_key`"상품번호"로 parse_download가 옥션 제외·A 중복제거 → 사용자 수기 합치기/중복제거 불필요. 다운로드 다회는 '신규만 추가'(merge가 A로 dedup) 또는 '전체 교체'(합본).
4. **배송비 = AI열 텍스트** ("무료"→0, "3,000"→3000). `_num` 콤마 허용으로 처리.
5. **N은 골든 대조 불가**(골든 N=#REF!) — consolidation 1193/1193 + 입력(판매가·배송비·base) 대조로 검증. N은 hapo(채널무관) 신뢰.

## 근거 / 검증
- consolidation 11+22+33→지마켓→A중복제거 = 사용자 중복제거 1193/1193.
- 골든 입력: 판매가 1193/1193·배송비 1193/1193·정산액(×0.825) 1189/1193·base 1170/1187(17 vintage).
