# ADR 0024 — 데일리 대시보드 품절 알림판 (발주 품절목록 → 재입고 reconcile)

- 날짜: 2026-06-16
- 상태: 채택

## 맥락
매일 발주작업(logistics-order Phase2)이 품절목록을 낸다. 사용자: 그 품절 상품을 데일리 대시보드 알림판에 띄워놓고, 재고가 보충되면 입고로그 남기고 자동 삭제(+수동 삭제), 안 들어오면 "MM월DD일부터 품절" 유지하고 싶다.

## 결정
1. **저장 = private data repo 영속**(비-PII): `history/stockout_board.json`(현재 알림판) + `history/restock_log.csv`(입고로그). 인박스(세션 휘발)와 달리 날짜를 넘겨 유지해야 하므로 영속. 관리코드·상품명·날짜·박스재고만 → PII 아님 → repo 가능.
2. **등록 = 발주 Phase2 품절목록 자동**(전건·시작일=그날·이미 있으면 유지). 1_파일처리 `_seed_stockout_board`.
3. **재입고 판정 = 박스재고(product_master '박스' 컬럼) > 0(양수 전환)** (사용자 확정). 현 품절건 박스재고 다 음수(초과판매/이월 drift) → 양수 전환이 깔끔한 보충 신호. 0=재고없음→유지. 데일리 대시보드 열 때 reconcile(라이브 product_master) → 회복 시 입고로그 append + 자동삭제.
4. **수동 삭제 = 로그 없이 제거**(사용자 확정).

## 근거 / 대안
- 박스재고 = '박스' 컬럼(박스내품 아님). 품절목록 현재고 = 박스재고 − 발주수량(박스단위) 검증(31-04-02: -19 − 39 = -58). 즉 품절목록=수요 대비 부족이라 절대 0 아님 → 회복 신호는 절대 박스재고 양수 전환(detect_transitions '0이하↔양수' 규칙과 일치).
- 대안(품절시점 대비 증가·최종재고 기준) 기각 — 사용자: 양수 전환이 현 데이터상 깔끔.
- reconcile는 lazy(데일리 대시보드 render 시 라이브 product_master 읽어 회복 판정) — 상품관리 업로드 페이지 훅 불요. `🔄 다시 읽기`로 캐시 clear.
