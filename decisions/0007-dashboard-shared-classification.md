# 0007 — 미분류 구분 분류: 공유 분류표 + 대시보드 분류 도우미

## 맥락
0006/dashboard.md는 대시보드 미분류 코드를 **발주 분류표(logistics_classification.csv)에 직접 넣지 말고 별도 UI**로 처리하라고 했다(발주 GATE 오염 우려). 그러나 별도 분류 저장소는 분류 소스가 둘로 갈려 유지보수 부담이 크고, 발주서출력업무 GATE A 자체가 이미 그 분류표에 코드를 추가하는 동일 메커니즘이다.

## 결정
미분류 구분은 **공유 분류표 그대로** 사용하되, 대시보드에 **분류 도우미 탭([🏷 구분 분류])**을 둔다. 미분류 관리코드를 상품명·매출·건수와 함께 보여주고, 사용자가 코드별로 음료/식품/선물세트를 지정 → logistics_classification.csv에 추가/덮어쓰기.

## 근거
- 분류표에 코드를 **추가**하는 것은 발주 GATE A 트리거를 **줄이기만** 하고 깨뜨리지 않는다(GATE A는 입력에 있으나 표에 없는 코드에만 발동).
- 코드별 사용자 지정이라 0006이 우려한 "전체 미분류 일괄 덤프"의 위험(쓰레기·오분류 코드 혼입)을 피한다.
- 분류 소스 단일화 → 발주·대시보드 양쪽이 같은 표를 공유, 한 번 분류하면 둘 다 반영.

## 잔여 리스크 / 가드
- 발주 흐름을 안 거치는 코드를 잘못 분류하면, 그 코드가 나중에 발주 입력에 들어올 때 GATE 없이 그 분류로 묵묵히 처리됨 → 코드별 수동 지정으로 완화(일괄 덤프 금지 유지).
- 분류표는 app repo(public)지만 관리코드·구분만 있어 PII 아님.

## 영향
- dashboard.md 전용 함정의 "분류 UI 별도" → "공유표 + 분류 도우미"로 갱신.
