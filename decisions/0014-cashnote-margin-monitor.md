# ADR 0014 — 캐시노트 채널 마진 모니터 추가 (모니터 전용)

날짜: 2026-06-11
상태: 채택

## 맥락
channel-margin-monitor에 캐시노트(KCD 마켓플레이스) 추가. 사용자가 상품관리 다운로드(`products_*.xlsx`) + 골든(`캐시노트_골든.xlsx`) 제공. 골든 정산식이 스마트스토어·식봄에 없던 변수(행사 차등수수료, 배송정책코드 조건부 배송비, 6%/7% 혼재)를 포함.

## 결정
1. **수수료 6%(×0.94) 단일 채택.** 골든 정산가(K)는 일반 0.93(7%)이지만, 다운로드 요율컬럼=6·천년경영 H식 0.94·골든 권장가(O) 0.94·헤더 "6%기준"이 다수 → 6%. 골든 K의 0.93은 시트 내부 불일치로 판단, 미채택. (사용자 확정)
2. **행사(이벤트) 차등수수료 무시.** 골든은 `캐시노트행사` 시트 상품번호에 0.88(12%) 적용하나, 우리 모니터엔 행사 목록 데이터 소스가 없음 → 전 상품 단일 수수료. 행사 목록 reference화는 필요 시 후속. (사용자 확정)
3. **배송비 = 배송정책코드 조건부(신규 config 축 `ship_fee_policy`).** 다운로드 Y열(배송 정책 코드)이 DVP212991이면 3000, 그 외(DVP447716)면 0. 식봄의 ship_fee_const 상수와 다름. 골든 J식(`IF(배송정책코드="DVP212991",3000,0)`)과 배송비 분포 484/33 정확 일치.
4. **표준 정산식 유지(2700·ceil).** 실택배비 2700 단일·권장가 100원 올림. 골든의 3000/3700·ROUND는 미채택(ADR 0013 식봄과 동일 — 스마트스토어 표준).
5. **상품번호 정수 정규화(`_pid`).** 엑셀 숫자셀 ID(46903.0)를 '46903'으로. 안 하면 hapo_multiplier(상품번호) 키 미스로 N 전건 1 오류.

## 근거
- 골든 513/513 전건 입력 대조: N·배송비 완전 일치, 매입가 461/463(2 vintage). 평균마진율 9.2%.
- 채널 추가가 "CHANNEL_CONFIG 한 세트 + 필요 시 parse 분기" 레시피로 유지됨(코드 4-tier·reference·페이지 공통 불변).

## 대안
- 7% 채택: 골든 정산식과 일치하나 다수 근거(요율=6·천년경영·권장가)와 배치. 기각.
- 행사 reference화 즉시 구현: 데이터 소스 없어 보류.
- 배송비 상수 3000(식봄식): DVP447716(33건) 무료배송 상품에서 마진 과소평가. 기각.

## 영향
- core 수정 → Reboot app 1회. ship_fee_policy·_pid는 타 채널 무영향(추가적·정수float만).
