# 0010 — KB 확장성 개선: 연결맵·상황별 분리·상태 단일원천

## 맥락
~10% 이관 시점에 에이전트 관점 KB 점검. 뼈대(상시읽기 바운드 + 워크플로우 lazy-load, 0004)는 건전. 그러나 셋이 확장에 약점:
1. 파일 연결성이 산문으로만 흩어져 공유 파일 변경 시 영향범위를 한눈에 못 봄.
2. pitfalls.md가 유일하게 무한정 크는 상시읽기 파일(11.2KB, 캐치업의 절반). 일부는 Streamlit/Excel 상황 한정.
3. 상태가 state 인덱스·workflows·로그 3곳 중복 → 드리프트(실제 이번 세션 ESM 카테고리표 '승인대기'↔커밋완료 불일치).

## 결정
- **manifest.md 신설**: 공유 reference→소비자 역색인 + 워크플로우 의존 + 등록채널 자산위치. 상시읽기 아님(공유자산 건드릴 때).
- **patterns.md 신설**: pitfalls에서 Streamlit/앱·Excel파싱 상황 한정 항목을 분리. 그 종류 작업할 때만 읽음. pitfalls는 프로토콜·환경·공통 데이터위생만(항상읽기 슬림).
- **state 인덱스 상태 = 통제어휘+포인터**: 상세 상태는 각 workflows/<name>.md 단일 소유. state 행은 운영중/진행중/이슈/개념doc + 링크만.

## 대안 (기각)
- 워크플로우 전용으로 pitfalls 항목 이사: 대부분이 진짜 횡단(플랫폼/포맷 거동)이라 단일 doc 귀속 부적합 → 상황별 patterns로 분리가 맞음.
- manifest 상시읽기: 캐치업 예산(22KB) 침범. 온디맨드로 충분.

## 영향
- 캐치업 상시읽기 세트 축소(pitfalls 슬림). 신규 교훈 라우팅: 프로토콜/환경→pitfalls, 앱/포맷 거동→patterns, 자산연결→manifest.
- 워크플로우 doc 헤더의 "전역 함정 pitfalls.md" 포인터는 유지(pitfalls 상단이 patterns/manifest로 재안내).
