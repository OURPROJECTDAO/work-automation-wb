# 0008 — 대시보드 온라인/전체 이익 정의 (택배비 = ERP 00-12 라인)

## 맥락
대시보드 다음 지표로 "이익" 요청. 온라인 팀 정의: 이익 = 매출 − 수수료 − 택배비 − 매입가, 이익률 = 이익 ÷ 매입가(분모가 매출 아님). 처음엔 수수료·택배비가 ERP에 없는 줄 알았으나, 실데이터(2026매출.xlsx = ERP 영업이익현황 그대로) 검산으로 다음을 확인.

## 발견 (실데이터 검산)
1. **수수료는 이미 판매금액에 반영**. 거래처 "오픈마켓- 스마트스토어" 행의 판매금액 = 판매가×(1−수수료). 별도 수수료 입력 불필요.
2. **항등식**: 판매금액 − 판매이익 = 판매원가(매입가). 스마트스토어 4105행 전수 일치. ERP 이익률 = 판매이익 ÷ 매입가(원가 분모) — 온라인 관례와 동일.
3. **택배비 = 관리코드 00-12 "C타입 택배비-3000" 라인**. 그 행의 수량 = 그날 그 거래처 송장(택배) 건수, 단위원가 3000, 판매단가 0 → 판매금액 0, **판매이익 = −(수량×3000)**. 즉 택배비가 음수 이익 라인으로 ERP에 이미 차감 반영. 온라인뿐 아니라 거의 모든 출고 거래처(나들커뮤니케이션 등 B2B 포함)에 존재.

## 결론 — 이익은 추가 입력 0, 기존 마스터(KEEP_COLS)만으로 계산
- 매출 = Σ판매금액(상품행, 00-12 제외)
- 송장건수 = Σ수량(00-12행) · 택배비 = 송장건수 × 단가
- 매입가 = 매출 − Σ판매이익(상품행)
- **이익 = Σ판매이익(상품행) − 택배비 = Σ판매이익(00-12 포함, @3000)**
- 이익률 = 이익 ÷ 매입가

## 결정 사항
- 적용 범위: **전체 거래처**(온라인 한정 아님). 그룹/거래처/일·월·연 집계.
- **택배비 단가 토글 3,000(ERP 입력값) / 2,500(보정값)**. 2500 선택 시 KPI·표·CSV에 "(보정)" 표기 + 경고 캡션. 보정 = 이익 + 송장건수×500. (전 채널 동일 단가)
- **상품별 이익 안 함**: 택배비가 개별 상품에 배분 안 됨(거래처×일 단위 lump). 일/월/연/거래처/그룹 집계까지만 정직하게.
- **이익 모드는 구분 필터 미적용**(전체 구분). 택배비 비배분이라 구분 부분집합 이익은 부정확.
- 매출 모드: 00-12 라인을 매출/상품 집계에서 **숨김**(판매금액≈0이라 영향 미미하나 "C타입 택배비" 상품 노이즈 제거).
