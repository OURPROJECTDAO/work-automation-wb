# 0021 — 채널 내 고객키/합포박스키: robust 마스킹 대응 (보이는 최소공통분모)

날짜: 2026-06-16 · 상태: 결정(설계, 미구현) · 관련: 0018(intelligence-layer)·합포 2시나리오(2026-06-16)

## 맥락
두뇌③ A/B·코호트 + 합포 그룹핑 정밀화(ship_alloc 과거판)를 위해 EasyAdmin 주문에 채널 내 **고객키(수령자명+전화)**·**합포박스키(수령자명+주소+발주일)** 를 비-PII 해시로 박제하려 함. 원본 .xls엔 풀 PII 있음(적재 parquet은 제거).
실측(2026-03~05, 25,822행)에서 **마스킹 정책이 시기에 따라 급변**(2026-03 51% → 04 0.5% → 05 0%, 이름·전화·주소 동시·전채널). 과거(2023~25)엔 또 다른 구간 가능성. 또 **G마켓·옥션·쿠팡은 안심번호(가상 전화)** 라 전화로 고객 식별 불가.

## 결정
1. **robust 키 = 보이는 최소공통분모만.** 마스킹/풀 무관하게 항상 보이는 조각으로 키 구성:
   - 이름 = 앞부분(끝글자 제외 — 마스킹 패턴 `ㄱㄱ*`)
   - 전화(실번호 채널) = 뒷4자리(`999-****-9999` 항상 보임)
   - 주소 = 시군구+도로명
   - 풀일 때도 같은 조각만 잘라 씀 → 시기 경계(정책 변경일)에서 재구매 추적이 안 끊김.
   - **풀 식별력 일부 포기 ↔ 시간 일관성 우선** (사용자 동의).
2. **채널별 레시피.** 안심번호 채널(G마켓·옥션·쿠팡)은 전화 제외(이름+주소). 주문자정보 없는 채널(알리·제이티·올웨이즈)은 수령자만.
3. **채널 내 키** (채널명 prefix). 채널 간 동일인 매칭은 후순위 별도 레이어.
4. **안전**: 해시(조각+솔트), 솔트=repo 시크릿, 원본 PII 즉시 폐기(송장그룹 해시 패턴).

## 대안 (기각)
- **풀 키(가용 조각 전부)**: 식별력 최대지만 마스킹 시기 경계에서 같은 고객이 다른 키 → 재구매 가짜 단절. 기각.
- **전화 단독 키**: 안심번호 채널 무력·이사 무관 안정이나 채널 3개 누락. 기각(이름+주소 보조 필수).
- **채널 간 통합 키 우선**: 엔티티 레졸루션 신뢰도 낮음·A/B엔 불필요. 후순위.

## 결과
- 충돌(다른 고객 같은 키) 위험 ↑ → 채널별 충돌률 검증으로 신뢰 등급. B2B 업소용은 재구매 多라 키당 주문 多가 정상(over-merge와 구분).
- 미구현. 다음 세션: 과거 통파일로 전구간 마스킹 확인 → robust 규칙 확정 → 충돌검증 → orders.py 키 → 재적재 → ship_alloc 연동. 상세 logs/2026-06/2026-06-16-customer-key-design.md.
