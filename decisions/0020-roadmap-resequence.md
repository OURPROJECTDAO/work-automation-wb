# ADR 0020 — 로드맵 재배열 (데이터 backbone 완성 → 신호 결합 단계)

날짜: 2026-06-15
상태: 확정

## 맥락
이력 엔진(Layer 1)이 사실상 완성. 사용자가 줄 수 있는 데이터 backbone 전부 적재됨:
매출 parquet(2023~·정산 진실)·EasyAdmin 주문 398K행(2023-03~2026-05·velocity·송장그룹)·
가격이력(수정로그·forward)·재고 스냅샷(forward)·**매입현황 66K행(2022-01~2026-05·실입고·실매입가)**.
모두 관리코드/일자로 조인 가능 → 두뇌가 **단일 신호 → 신호 결합** 단계로 전환 가능.

## 결정
intelligence-layer 로드맵을 ROI·데이터 신선도·구현 난이도 기준 재배열(정본=systemmap.json):

1. **[next ★1] 두뇌① 강화 — 2C 조기경보 + velocity 가중.**
   기존 침식 v1은 반응형(이미 침식된 것). ① 매입현황 실입고단가↑ ∩ master 미수정 → 침식 *전* 예방 경보,
   ② 침식폭×velocity=월 손실액 정렬. 페이지(8_마진침식.py) 존재 → 가장 가볍고, 매입현황 신선할 때 즉시 검증.
   (기존 later '두뇌①+ velocity 가중'을 여기로 흡수.)
2. **[next 2] 두뇌② 입고·품절 예측(planned→승격).**
   현재고+velocity+리드타임(1c) 충족 → 지금 가능. upload-monitor 반응형→예방형. 1c 리드타임 동반.
3. **[planned 3] 두뇌③ 채널 가격 A/B(next→강등·강화).**
   채널별 실현마진(P2 송장실배분+정산+매입현황)×velocity=진짜 단위경제. 신규 페이지·채널명 정규화 선행.
4. **[planned 신규] 상품 360도 카드.**
   두뇌 3종 후 통합 뷰 — 관리코드 1개 → 매입가추이·채널마진/velocity·재고/소진·침식경보 한 화면.

나머지(1d listing 스냅샷·1e 행사로깅·시장지능·퍼널·세트보정·상품택배비·2023여름갭·데이터현황 2단계 UI)=later 유지.

## 근거
- 2C는 매입현황 적재로 *방금* 열린 능력 — 신선할 때 검증 ROI 최고, 반응형→예방형 질적 도약.
- 두뇌②는 입력(현재고·velocity·리드타임)이 전부 충족된 상태라 더 미룰 이유 없음. upload-monitor와 자연스러운 짝.
- 두뇌③ A/B는 가치 크지만 신규 페이지·채널 정규화·준실험 교란(행사) 부담 → 데이터 결합 능력 검증 후가 안전. 단 실현마진 결합으로 목표는 상향.
- 상품 360도 카드는 두뇌 3종이 있어야 의미 → 후행 통합.

## 영향
- systemmap.json intelligence-layer.roadmap 전면 재배열 + upload-monitor 두뇌② 연계 명시.
- 다음 착수 = 두뇌① 강화(2C 조기경보). 8_마진침식.py + margin_erosion.py 확장(core → Reboot).
