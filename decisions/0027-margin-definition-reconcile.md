# ADR 0027 — 마진 정의 정합: cmm(노출가 target) ↔ 두뇌④(매출자료 realized), 기준마진율 변경은 Δ가산

날짜: 2026-06-19 · 상태: 확정 · 연계 ADR 0026(두뇌④)·0016(채널마진모니터)

## 맥락
두뇌④ 권장 기준마진율을 채널마진모니터(cmm) 기준마진율(baseline_margin.csv)에 반영하려는데, 두 "마진"이 같은 축인지 검증 필요.

## 발견 (실데이터 분해 — 15-04 스팸클래식340 캐시노트)
- **공식 뼈대는 동일**: 마진 = (정산액 − 매입가 − 택배) / 정산액.
- **정산액 정의가 다름** ← 핵심:
  - cmm 정산액 = **노출가(리스팅 표시가) × (1−수수료) + 배송비×정산계수** → "표시가에 팔린다면"의 **forward 추정**.
  - 두뇌④ 정산액 = **매출자료 실제 정산금액** → 쿠폰·즉시할인·행사·믹스 다 반영된 **realized**.
- 매입가: product_master 3,400 ≈ 두뇌④ 역산 3,449 → 거의 같음(갭 아님).
- 택배: cmm 플랫 2,700/박스내품(=135/낱개) vs 두뇌④ ship_alloc 실측(98/낱개) → 약간 다름.
- 결과: cmm 타깃 **8.56%** vs 두뇌④ 실현 **5.74%**, **실현갭 2.82%p** (대부분 정산 차이 = 실제 116/낱개 덜 받힘).
- 갭은 정상 — 사용자 확인: 택배 위치·할인쿠폰·기준마진율대로 즉시 안 고치고 두는 경우 등으로 늘 생김.

## 결정
- 두뇌④ 권장(realized)을 cmm 기준마진율(target)에 **절대값으로 쓰지 않는다.**
  - 이유: 방향 역전 위험. realized 6.1%를 타깃에 절대 기입 → cmm는 "노출가에서 6.1%"를 목표로 권장가를 낮춰 잡고 → 실제 실현은 6.1% − 갭2.82% ≈ 3.3%로 **하락**(현재 5.74%보다 낮아짐). 올리려다 내리는 꼴.
- 대신 **현 기준마진율 타깃 + Δ** 를 기입. Δ = 권장 − 현재 (= 표의 "변화" 컬럼, 실현공간 변화량).
  - 예: 캐시노트 타깃 8.56% + (+0.36%p) = 8.92%. 타깃을 Δ만큼만 밀어 → 실현도 ~Δ만큼 이동.
  - 전제: 작은 절반스텝에선 할인갭 ≈ 일정. 큰 변동엔 약해짐(주의).
- 기준값(baseline 행) 없는 셀은 Δ 더할 대상이 없으니 **기록만**.

## 구현 (page 13, 페이지-only)
- "기준마진율 변경" 토글 = 타깃+Δ 가산(절대 덮어쓰기 폐기). 미리보기에 **기준 → 새기준**.
- 표에 **기준마진율(현 타깃)** 컬럼 — baseline_margin.csv 채널별 조회(load_baseline, GITHUB_PAT).
- 채널 라벨 → baseline 컬럼 매핑(_BCOL): ESM(G마켓·옥션)→ESM, 나머지 동일. 매핑 외(제이티 등) 건너뜀.
