# ADR 0023 — 데일리 대시보드: 당일 점검 승격 + 세션 자동 파일 인계

- 날짜: 2026-06-16
- 상태: 채택

## 맥락
마진침식 탭D(당일 점검)는 천년경영 output·송장출력을 **매번 수동 업로드**해야 했다. 이 둘은 사용자가 매일 파일처리에서 이미 생산·출력하는 자료다. 사용자: 데일리 대시보드 = 매일 반복 업무 산출물로 당일 인사이트를 보는 곳(당일 점검의 확장판). 탭D를 그 첫 브릭으로 독립.

## 결정
1. **당일 점검(탭D)을 독립 페이지 `데일리 대시보드`로 승격**(네비 첫 그룹·지도·로드맵 아래). 마진침식은 3탭으로 축소.
2. **세션 자동 파일 인계**: 파일처리(오픈마켓→송장출력·천년경영→output) 산출 시 `core/intelligence/daily_inbox.py` 세션 인박스에 push → 데일리 대시보드가 재업로드 없이 사용. 슬롯별 수동 갱신 override("그 시점 파일로"). 상품관리는 reference 라이브(인계 불요).

## 근거 / 대안
- **세션(st.session_state) 인박스, 휘발성**: 송장출력에 PII(수령자·주소·송장번호) → 전역 규칙상 디스크/repo 미저장. 세션 메모리만이 안전. 대안(영속 캐시)=PII 노출이라 기각. 같은 작업 세션이면 자동, 리부트/새 세션이면 수동(수용 가능 — 일일 업무는 보통 한 세션).
- **core는 streamlit 비의존**: daily_inbox는 session_state를 인자로 받는 순수 함수(키 상수만 소유). 슬롯 키 단일 진실원천으로 생산/소비 드리프트 방지.
- 송장출력 = 오픈마켓 ★★송장(generate_invoice_xlsx) = 탭D 0615 검증 형식과 동일(판매처[3]·수령자[5]·주소[6]·상품명[7]·송장번호[9]) → 자동 인계 호환 확정.

## 영향
- 신규: 0b_데일리대시보드.py · daily_inbox.py. 수정: 1_파일처리.py(push 훅)·8_마진침식.py(탭D 제거·dm import 제거)·streamlit_app.py(nav).
- daily_margin.py 무변경(탭D와 공유). systemmap daily-dashboard 노드 신설(meta n).
- 새 core 모듈 → 첫 배포 안 보이면 Reboot app 1회.
- 다음(확장판): 추가 데일리 인사이트(품절/발주/업로드 등 반복업무 결합).
