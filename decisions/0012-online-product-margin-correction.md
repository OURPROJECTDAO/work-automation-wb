# ADR 0012 — 온라인 상품별 마진(추정) + 채널 보정계수

## 상태
채택 (2026-06-10)

## 맥락
집계 매출자료(parquet 9컬럼)엔 택배비(00-12)가 주문/상품과 분리돼 있어 상품별 순마진을 정확히 못 냄. 게다가 가격 산정 때 실택배비를 단가에 녹여 상품 판매이익에 회수분이 섞임. 단위원가/판매단가는 KEEP_COLS에서 잘려 parquet에 없음(도출은 가능하나 행평균).

## 결정
상품별 마진을 **추정**으로 제공. 핵심은 택배비를 합포수량으로 상품에 배분:
- 추정송장 = 수량 ÷ (합포수량 × 내품수[=product_master 박스내품])
- 추정택배비 = 실택배비(3000/2500 토글) × 추정송장
- 상품 마진율 = (판매이익 − 추정택배비) ÷ 매입가,  매입가 = 판매금액 − 판매이익
- 단위원가/판매단가는 판매금액·판매이익·수량에서 도출(재적재 불필요).

**온라인 한정**: 택배비(00-12) 행이 있는 거래처만 대상 — 데이터에서 자동 판별(상호명별 00-12 유무). 별도 페이지 대신 **같은 대시보드의 별도 탭**(데이터로더 공유).

**낙관 편향 보정(핵심)**: '최대합포로만 팔렸다' 가정이라 추정송장<실제송장 → 택배 과소·마진 낙관. **채널 보정계수 k = 실제송장(Σ00-12) ÷ 추정송장**을 곱해 실제 택배비 총액에 정확히 수렴시킴. k는 보고 중인 기간·선택거래처의 실제 데이터에서 매번 자동계산(유지보수 0, 항상 실제와 합치). 토글로 (a)순수추정/(b)보정 선택, 기본 보정.

## 결과/검증
- 스마트스토어 5월: 미보정 6.89% / 보정 5.98%(=채널 실제 이익률과 정확 일치, 택배 6,170,000 = 실제송장 2,468×2,500). k=1.115.
- 구현: app/pages/3_대시보드.py '💰 상품마진(온라인)' 탭(commit 2f44d074). load_sales에 합포수량·박스내품 컬럼 추가. 페이지 변경이라 Reboot 불필요.

## 한계
- 짧은 기간은 k 출렁임(월단위 권장). 단위 단가는 행평균(원본값 필요 시 재적재).
- 정확 배송정산 마진은 주문 원본(스크래핑/API, 로드맵 후반) 필요.
