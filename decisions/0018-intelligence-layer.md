# ADR 0018 — 지능 레이어 (이력 엔진 + 두뇌: 마진·재고·A/B)

2026-06-12. 상태: 채택(설계확정·미구현, 직접 실행 다음 세션). 상세 plan/데이터 카탈로그 = workflows/intelligence-layer.md.

## 맥락
- 시스템에 **관찰**(대시보드·channel-margin-monitor 8채널·upload-monitor)과 **실행**(가격변경 7채널·상품등록·발주·송장)은 이미 구축됨. 그 사이 **두뇌**(진단·추천·측정)가 빈칸 — 사람이 대시보드를 보고 판단→가격변경→다음달 눈대중으로 효과 확인.
- 사용자 목표: 매출/마진 증대 A/B, 적절한 마진율 제시, 품절/입고 알림, AI.
- 데이터 부족이 문제인 줄 알았으나 — 실물 9종 확인 결과 **이미 다 있고 안 모으고 있었을 뿐**. 매일 웹앱으로 추출하는 자료가 덮어쓰기/미보관일 뿐.

## 결정
2층 구조 + 루프(관찰→진단→추천→실행→측정→학습):
- **Layer 1 이력 엔진(메모리)**: 과거 소급 4종(수정로그·매입현황·EasyAdmin 주문·대시매출) + 현재 앵커 1종(상품관리 스냅샷) → private repo(work-automation-data) 날짜 파티션. 대시보드 parquet 패턴 재사용.
- **Layer 2 두뇌(결정)**: ①마진 침식·제시 ②입고·품절 예측 ③채널 가격 A/B. 실행은 기존 actuator 재사용.
- 첫 브릭 = **수정로그 3년 적재**(가장 작고, 적재 즉시 3년 마진 이력 소급 → day-one 가치).

## 핵심 사실 / 정정 (사용자 명시 — 헷갈리면 안 됨)
- **(A) 정산 진실원천 = 매출자료**(천년경영 매출 업로드 정본 = 대시보드 영업이익현황 소스). 판매금액=정산. EasyAdmin/erp 정산은 raw 상류(매일 추가정제 대상). → 채널 정산내역서 별도 파싱 불요. "더 정확할 필요 없음".
- **(B) 택배 실배분 = 송장번호 그룹**(EasyAdmin 주문 export에 송장번호 컬럼 존재). 같은 송장번호=한 박스(합포). 대시보드 온라인 마진 탭 추정송장·보정계수 k를 실제값으로 대체 → 상품별 마진이 추정→정확. 별도 송장출력 파일 불요.
- **(C) 매입가 두 출처**: master 매입단가(수정로그가 변경 기록) vs 매입현황 실입고 단가. 마진모니터/실현마진=master 기준. 차이=신호(매입현황↑·master 미수정→장부마진 과대).
- **(D) 마진모니터 정산식**(스마트 ×0.94 등)=*앞으로 가격 정할 때* 추정용. *실현* 마진=매출자료(정산)+수정로그(매입가)+송장(택배).
- **(E) 스케줄러 없음**(Community Cloud) → 알림=GitHub Actions cron(repo diff→webhook) 또는 인앱 스냅샷 diff.

## 대안 기각
- 채널별 실제 정산내역서 파싱(원안 Phase 2) → **기각**: 매출자료가 이미 정산 진실(A). 한 단계 통째 제거.
- 매일 forward 스냅샷만으로 마진 이력 축적(몇 달 대기) → **축소**: 수정로그가 3년 변경 이력 소급 제공(가격 측). 스냅샷은 *현재 재고 앵커*로만(재고 절대값은 흐름누적 오차).
- velocity 소스로 대시보드만 사용 → **보강**: EasyAdmin 주문이 채널 명시·주문해상도·판매가(가격점)·송장으로 우월(2025+ 소급).

## 결과 / 영향
- 대시보드 온라인 상품마진 탭(ADR 0012, 추정)은 본 워크플로우의 실측 송장 실배분으로 **대체 대상**.
- channel-margin-monitor에 **시간축** 부여(현재 마진→마진 추이·침식).
- 신규 core 모듈 `core/intelligence/`(price_history·ship_alloc 등), 신규 private repo 파티션(§4 workflow doc).
- A/B는 준실험(전표 고유ID 없음·수동 다운로드)·계절·행사 교란 → 행사 로깅(forward) 보정.

## 단계
1 이력 토대(수정로그 적재[첫]·상품관리 스냅샷·리드타임 조인·listing 스냅샷 보관·행사 로깅 시작) → 2 진짜 마진(매출자료+수정로그+송장 실배분; 정산내역서 불요) → 3 시장지능(경쟁가 크롤링, 별도 트랙) → 4 퍼널(가능 채널 통계).
