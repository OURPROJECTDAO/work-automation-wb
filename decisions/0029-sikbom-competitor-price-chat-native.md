# 0029 — 식봄 경쟁가 조회 = 채팅 네이티브 + 코드화 크롤 스크립트 (nadl과 분리)

## 맥락
- 2026-06-24 식봄(foodspring) GraphQL 경쟁가 크롤링 실증 성공. 운영 형태를 정해야 함.
- 식봄은 마진모니터/송장/행사기획엔 기존 채널이나, **경쟁가 조회는 신규 capability**.

## 대안
- **A. nadl 패턴**: 페이지(시장가매칭)+주기 스냅샷(parquet)+관리코드 1회 사람확정 매핑 영속.
- **B. 채팅 네이티브 ad-hoc + 코드화 크롤 스크립트**.

## 결정 = B
- **식봄 ≠ nadl (본질 차이)**:
  - nadl = B2B 카탈로그를 **통째 주기 스냅샷**·우리코드 1회 매핑·정적·대량 → 페이지 적합.
  - 식봄 = **라이브 마켓플레이스**. 같은 상품을 **판매자별 다른 이름**으로 등록(예 "명가 꽈배기 참깨 500g 1개" vs "명가꽈배기 참깨맛 500g (개별포장) 간식 옛날과자"). 우리도 한 판매자.
  - 식봄 업무는 **질의형**(특정 상품번호/상품군을 불러줄 때) + **판단형**(같은 상품이냐·규격 매칭은 의미매칭 필수=AI). 미리 다 긁어둘 가치 없음 → 페이지·주기 스냅샷 부적합.
- **분업(토큰 경제)**:
  - 기계층 = **스크립트** `tools/sikbom_price_lookup.py`: 세션 쿠키·GraphQL POST(역공학 1회분 쿼리 박제)·파싱·식봄 정산마진(최저/우리/3%하한/BEP). 매 세션 재탐색 불필요·토큰 0.
  - 판단층 = **AI 채팅**: 판매자별 이름 차이 넘어 같은 상품 판별·규격 동일성·제안·표 정리. 못 줄임.
- 산출물(xlsx 등)은 **요청 시만**(기본은 채팅으로 끝냄).

## 사용
- `python sikbom_price_lookup.py --keyword "명가꽈배기" [--cost 매입가] [--our 현재가]` → 판매자별 표 + 정산마진.
- 상품번호 주어지면: listing_sikbom→관리코드·product_master→매입가(단위별) 먼저 해소 후 키워드로 조회.

## 결과 / 유지
- 매 질의 토큰 대폭 절약(역공학·수기 curl/parse 반복 제거).
