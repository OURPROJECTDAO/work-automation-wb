# 0003 — 발주서출력업무: 2-phase + 에러 게이트 모델

## 맥락
원본 물류팀프로그램v5_2 xlsm은 16단계 매크로를 사용자가 버튼 하나씩 수동 클릭.
중간중간 에러 확인 구간(미분류 코드, 빈칸 검사)이 있어 일부러 단계를 쪼갠 구조.
웹앱 이관 시 이 "사람 확인 구간"을 어떻게 재현할지가 핵심 갈림길.

## 검토한 대안
- A. 충실한 16버튼 단계별: 친숙하나 클릭 과다.
- B. 2-phase 묶음 + 검증 게이트: 정제+분류 / 재고대조+출력 2단계, 미매칭만 게이트로.
- C. 원샷 + 사전 일괄검증: 실행 전 모든 미매칭 스캔.

## 결정 (B)
2-phase + 에러 게이트:
- Phase1(정제·셀나누기·구분분류·중복제거·규격) → GATE A(구분 미분류 코드).
- Phase2(자리바꿈·정렬·통합·재고대조) → GATE B(낱개 원코드 미매칭).
- 에러 없으면 클릭 2번으로 끝. 있으면 그 지점에서 멈추고 기준데이터에 추가 후 재실행.

## 근거
16단계 "확인 구간"이 결국 전부 미매칭 코드 채우기였음. 앞단 게이트로 모으면 사람 개입 1~2회로 축소.
단계별 눈 확인 습관은 Phase 구분 + 메트릭 표시로 일부 유지.

## 결과 / 영향
- 미매칭 발견 시 기준데이터(멸치쇼핑 분류표 / 낱개처리목록)에 자동 추가 → 다음번 자동 통과.
- 발주자료 아카이브는 Phase1 종료 시 별도 저장 (원본 8열, 중복제거 전).
- 상품관리(재고)는 매일 갱신 → 연동데이터관리 페이지 + 타임스탬프 확인.
- 기준데이터(분류표/낱개목록/규격파일)는 고정 → 기준데이터관리 탭.
